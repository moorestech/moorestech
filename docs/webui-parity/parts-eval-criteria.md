# Web UI ↔ uGUI パーツ別視覚一致 評価基準・評価方法

正本uGUIスクリーンショットとWeb UI実装を「並べても見分けがつかない」状態へ収束させるための採点基準。
知覚初稿はcodex(gpt-5.6-sol)による全100マス走査の差分棚卸し（`diff-inventory-2026-07-17.md`）に基づき、
数値は本リポジトリでの画素実測により裏取りした（visual-criteria-cross-eyesプロトコル準拠）。

- **[実測]** = Claude側で独立に画素実測して確定した値（正）
- **（要確認）** = codex観測のみで未裏取り。影・発光・フェード境界など単一閾値で確定しにくい値
- 本書には「現状」の数値を書かない。採点時の現状値は毎回 `parity-check.py` と画素実測で取り直すこと（過去値の引き写し防止）

## 前提ハーネス

- 正本: `docs/webui-parity/reference-player-inventory-3270x1844.png`（3270×1844）
- 現状撮影（`moorestech_web/webui/`で実行、3270×1844・モックDEMO・レシピ1件選択）:
  ```bash
  CAPTURE_VIEWPORT_W=1635 CAPTURE_VIEWPORT_H=922 CAPTURE_OUT=out/current.png \
    ./node_modules/.bin/tsx e2e/capture-eval.ts
  ```
- パーツ別同一座標クロップ＋50%ブレンド＋difference＋細部サブクロップ:
  ```bash
  python3 e2e/crop-parity-parts.py \
    --ref ../../docs/webui-parity/reference-player-inventory-3270x1844.png \
    --cur out/current.png --out out/crops --overlay --subcrops
  ```
- 座標は特記なければ元画像（3270×1844）上の絶対px。
- パネル本体境界の測定規則: 対象帯で「max(R,G,B)<110の画素割合が50%を超える」最外行/列（以下「暗色50%規則」）。
  フェード・影で曖昧な下端はこの規則値に対し±6pxまで許容。

## 既定許容差

- 主要bbox: 各辺±3px ／ その他の座標: ±1.5%pt ／ 線厚: ±2px
- 対応点の色: 5×5px中央値でRGB各チャンネル±15

# 2. 評価方法

## 2.1 再実行手順

1. 状態パリティ（§3.1）を満たした状態で3270×1844の現状画像を撮影する（前提ハーネスのコマンド）。
2. 正本・現状がともに3270×1844であることを確認（サイズ不一致ならcrop-parity-parts.pyが失敗する）。
3. `crop-parity-parts.py --overlay --subcrops` で4パーツのref/cur/blend/diffと細部サブクロップを生成する。
4. ref/curを等倍で並べ、外寸→空洞→線数→枠層→角→透過→色→付属要素→文字の順で確認する。
5. blendで二重輪郭（位置ズレ）を確認する。
6. differenceで位置ズレ以外の色・発光・影・文字差を確認する。
7. サブクロップを200–400%表示で確認する。
8. §2.2の色テーブルを5×5px中央値で測定する。
9. 各軸を採点し合計する。

## 2.2 機械的な色ピック

単一画素はAA・ノイズの影響を受けるため、指定点中心の5×5px中央値を使う。文字・アイコンが侵入した場合は同一面内で±10pxまで移動し記録する。1–2pxの線は5×5中央値でなく線中央の3点を直接測る。

| 対象 | 元画像座標 | クロップ内座標 | 正本RGB [実測] |
|---|---:|---:|---:|
| 背景・上中央 | (1635,100) | — | (143,120,96) |
| 背景・左中段 | (40,900) | — | (139,95,49) |

## 2.3 重ね合わせ判定

### 50%ブレンド
境界が一致すればblend上で単一輪郭、ズレていれば正本色と現状色の二重輪郭になる。

- 主要bboxの二重輪郭間隔が各辺3px以下
- スロット列の周期的な二重線が2px以下
- タイトル・ボタン・スクロールバーの中心ズレ3px以下
- 同一位置の線が色差だけで二重に見える場合は、位置点を与えた後に色軸で別途減点

### difference画像
除外マスク（§3.2）外で `Dmax = max(|ΔR|,|ΔG|,|ΔB|)` を集計する。

- 主要枠の±3px帯の外に連続した強差輪郭（Dmax>40）がない
- 面色領域は対象面の90%以上が Dmax≤15
- 発光・影領域では正本境界から6pxを超える強差がない
- 全画面一括の差分率だけでは採点しない（背景・モック差が細部差を隠すため）
