# Web UI uGUI Visual Parity Design

## Goal

インベントリ・クラフト画面のWeb UIを、uGUI時代の正本が持つ半透明なゲーム内HUD表現へ近づける。機能やデータ内容は変更せず、パネル、罫線、選択枠、ホットバー番号、装飾の視覚差をCSS中心で解消する。

## Scope

- `GamePanel` の背景、外枠、見出し罫線
- クラフト中央パネルとレシピ選択枠
- クラフトフィールド右下の階段三角装飾
- 見出し境界中央のダガー・ダイヤ装飾
- 選択中スロットの直角コーナーと多層グロー
- ホットバー番号の位置と囲い
- 現状だけに存在するキャラクター由来の誤認要素の削除
- Playwrightキャプチャによる反復評価

アイコン絵柄、アイテム名、個数など、モックデータと正本データの内容差は対象外とする。

## Chosen Approach

CSS疑似要素を主体とし、疑似要素だけで表現できない場合に限って意味を持たない装飾要素を最小限追加する。

画像スライス中心の実装は解像度追従と調整負荷が大きいため採用しない。SVG中心の実装は今回の単純な直線・段差・菱形装飾に対して構造が過剰になるため採用しない。

## Component Design

### GamePanel

- 背景はゲーム画面が連続して見える暗色半透明とする。
- 広いぼかし影を廃止し、短く硬いピクセル影を使う。
- 角丸を廃止し、外枠と内周線からなる直角の二層構造にする。
- タイトル罫線は閉じたカードの一辺ではなく、上下に独立する複層水平線として描く。
- 水平線は左右端で背景へ減衰させ、均一な終端を残さない。
- 見出し用途に応じて、持ち物用とCRAFT RECIPE用の間隔・線幅を調整可能にする。

### Recipe Viewer

- 中央クラフトパネルも大きな外枠と内周線の二層構造にする。
- クラフトフィールド右下へ、複数の明暗層と底辺帯を持つ階段状三角装飾を置く。
- 境界線中央へ、左右の内向きダガー、中央の多層菱形、段階減衰する複層線を一体で描く。
- 正本にない中央の独立カード、キャラクター画像、暖色オブジェクトは描画しない。

### ItemSlot

- 通常枠は直角とし、暖色寄りの半透明な複層線を使う。
- 選択枠は暗外縁、青帯、シアン芯、内側濃紺線、外側グローの最低五段階とする。
- 四隅は角丸ではなく、直線部より明るい直角L字コーナーキャップにする。
- 選択状態を示す既存の `data-selected` 契約は維持する。

### HotbarPanel

- 番号容器はスロット内左上から撤去し、各スロットの外側上方中央へ独立配置する。
- 番号容器とスロット上辺の間に空洞を設ける。
- 各番号の中心を対応スロットの中心へ揃える。
- 内蔵タブや左上切欠きに見える形を残さない。

## Visual Acceptance Criteria

Playwrightを正本と同じviewport、同じUI状態、同じcropで撮影する。座標許容は正本crop比で±1.5ポイント、線厚は±2px、主要bboxは±3px、色は同一背景・同一座標でRGB各チャンネル±15とする。

以下をすべてMUSTとし、合計90/100以上を合格とする。

1. 透過・非破壊ガード（15点）
   - 背景模様がパネル越しに連続して見える。
   - 不透明な固定暗色面、黒ページ背景、広いぼかし影がない。
   - 現状画像中央の暖色誤認要素が存在しない。
2. 持ち物見出し罫線（12点）
   - 上下に独立した約8pxの複層線があり、線間に十分な空洞がある。
   - 左縦辺、角丸、均一終端がない。
3. CRAFT RECIPE罫線（12点）
   - 上下とも明暗を持つ複層線で、端が減衰する。
   - 大型カード、左縦辺、角丸がない。
4. 中心・クラフト枠（12点）
   - 外枠と内周線の二層が見える。
   - 左下を含む全角が直角で、影は短く硬い。
5. ホットバー（12点）
   - 番号がスロット外上方中央にあり、番号とスロットの間に空洞がある。
   - 番号中心とスロット中心が±3px以内で揃う。
6. 右下三角装飾（10点）
   - クラフトフィールド右下内側に、複層の階段三角と底辺帯がある。
7. ダガー・ダイヤ罫線（12点）
   - 左右線、内向きダガー、中央多層菱形が一体となっている。
   - 全幅均一な単線や現状だけの青帯がない。
8. 選択枠（12点）
   - 最低五段階のネオン層と直角L字コーナーキャップがある。
   - 均一な水色角丸borderではない。
9. 文字・画素感・存在パリティ（3点）
   - 正本に近い低解像度ゲーム字体、灰色アンチエイリアス、段階影を保つ。
   - 正本にない閉じたカード、縦辺、均一セパレータ、内側番号タブを残さない。

## Iteration and QA

既存のPlaywright採点用キャプチャを使い、以下の順で差分を潰す。

1. 透過と誤認要素削除
2. 持ち物・CRAFT RECIPEの二重線と端の減衰
3. 大枠と内周線
4. ホットバー番号、右下三角、ダガー・ダイヤ
5. 選択グロー
6. 文字と存在パリティ

各反復で等倍cropを保存し、MUSTごとに再採点する。最初の比較で問題が見つからない場合は検査不足とみなし、線数、枠の層、空洞、付属要素の位置を個別に再確認する。

## Verification

- Web UIの型チェックとproduction build
- 関連Vitest
- 関連Playwright E2E
- 採点用Playwrightキャプチャ
- 正本との等倍crop比較
- 全MUST達成かつ90/100以上

`.cs` ファイルは変更しないため、Unityコンパイルは不要とする。Unity固有YAMLと `.meta` は編集しない。
