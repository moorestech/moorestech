# 建設メニューのカテゴリ化（Satisfactory風）設計

日付: 2026-07-21
対象ブランチ: feature/machine-ui-tabs から分岐（または継続）

## 背景・目的

現在のWeb UI建設メニュー（`moorestech_web/webui/src/features/buildMenu/`）は全エントリをフラットな10列グリッドで並べるだけで、73ブロック+車両+ツール+ブループリントが無分類で混在している。これをSatisfactoryのビルドメニュー同様の「左サイドバー大カテゴリ + サブカテゴリ見出し付きグリッド + 検索 + ホバー詳細」へ再設計する。

## 決定事項（ユーザー承認済み）

- カテゴリ階層は**2段**（サイドバー大カテゴリ + グリッド内サブカテゴリ見出し）
- **検索ボックスを実装する**（全カテゴリ横断・名前部分一致）
- ホバー詳細は**パネル内固定プレビュー欄**（機械レシピ選択タブ様式の前例一致。カーソル追従カードは作らない）
- カテゴリ値はマスタデータが正本。`category` は**必須化**し全ブロックへ一括付与（optional+未分類フォールバック禁止 / PR978原則）
- カテゴリ・サブカテゴリの**表示順は所属エントリの最小 sortPriority から導出**（順序定義用の新マスタは作らない）
- 非ブロックエントリは**エントリ種別から固定カテゴリを導出**（マスタ拡張しない）

## 1. マスタデータ設計

### 1.1 スキーマ変更（VanillaSchema/blocks.yml）

- 既存 `category`（optional string）を **required に変更**
- `subCategory`（required string）を追加

両方とも自由文字列。カテゴリの一覧・順序・表示名はデータから導出するため、カテゴリ定義用の別マスタ・enumは作らない。付与漏れはmooresmasterのロードで必須違反として大声で失敗する。

## 2. 配信契約の変更

- `BuildMenuDtos.cs` の `BuildMenuEntryDto` に `Category` / `SubCategory`（required string）を追加
- `BuildMenuEntryDtoFactory.cs` がブロックはマスタ値、非ブロックは種別導出値を設定
- あわせてカテゴリ内順序決定用の `SortOrder`（int）を追加（ブロック: sortPriority、非ブロック: 種別ごとの固定オフセット + 列挙順）
- webui側 `bridge/contract/schemas/buildMenu.ts` の zod スキーマへ `category` / `subCategory` / `sortOrder` を必須追加
- `WireContractTest`（C#）とmockホストのフィクスチャを同時更新

## 3. Unity側（Client.Game / Client.WebUiHost）

- `BuildMenuEntry` にカテゴリ情報を追加（マスタ値をそのまま保持）
- `BuildMenuEntryCatalog` は列挙ロジック変更なし（unlock/デバッグ表示の挙動は現状維持）
- uGUI側ビルドメニューは残置のまま非対応（UIはWeb移行完了方針）

## 4. webui設計

### 4.1 レイアウト

```
┌─────┬──────────────────────────────┐
│特殊  │ [検索入力]                    │
│生産  │ ┌ 詳細プレビュー（固定高）─┐   │
│電力  │ │ アイコン 名前 建設コスト │   │
│物流  │ └──────── FadeRule ──────┘   │
│輸送  │ サブカテゴリ見出し(muted)      │
│ …   │ [slot][slot][slot]…          │
│     │ サブカテゴリ見出し             │
│     │ [slot][slot]… (縦スクロール)  │
└─────┴──────────────────────────────┘
```

- 外枠: 既存どおり `GamePanel variant="default"` + `title`。パネル位置・開閉トリガは現状維持
- 左サイドバー: `ModeSwitch` を**縦向き**に使用（汎用I/Fのまま、縦並びは利用側CSSで実現）。選択中カテゴリは `data-selected`
- スロット: 既存 `BuildMenuSlot`（`SlotFrame`系）を継続。ホバーは `onHoverChange` で親へ通知
- サブカテゴリ見出し: `--text-muted` のラベル + `FadeRule`。独自装飾は作らない
- 詳細プレビュー: 機械レシピ選択タブと同様の**固定高**上段。ホバー中エントリを優先、無ければ選択中エントリ、どちらも無ければ `--text-muted` の案内テキスト。内容は「アイコン（`GameIcon`系）+ 名前 + 建設コスト（既存tooltip文字列の構造化: 名前行 + 素材xN行）」
- 既存のMantine `Tooltip` によるホバーテキストは詳細プレビューに置き換えて廃止

### 4.2 状態と挙動

- 選択中カテゴリ: `useState`。初期値は先頭カテゴリ
- カテゴリ順: エントリ配列の `sortOrder` 最小値昇順で導出。サブカテゴリ順も同様。**エントリが1件も無いカテゴリはサイドバーに出さない**（unlock進行で自然に増える）
- 検索: 入力非空の間は**全カテゴリ横断**で `label` 部分一致（大文字小文字無視）。サブカテゴリ見出しは維持し、カテゴリをまたぐ場合は「カテゴリ名 / サブカテゴリ名」の複合見出しで区切る。**検索中はサイドバーを無効化（グレーアウト・クリック不可）**し、どちらのフィルタが効いているか曖昧にならないようにする。検索クリアで元のカテゴリ表示へ戻る
- クリック挙動（左クリック選択・blueprint右クリック削除等）は現状の `select` アクション契約のまま変更しない

## 5. エッジケース

- **付与漏れブロック**: スキーマrequired化によりマスタロードで即失敗（無言の未分類落ちなし）
- **保存BPの動的増減**: 種別導出のため常に「ブループリント」カテゴリに出る。BP0件かつ他エントリ0件のカテゴリは非表示規則で消える
- **検索0件**: グリッド領域に `--text-muted` の「該当なし」テキスト
- **未解放進行初期**: 解放済みが1カテゴリしか無い場合もサイドバーは1項目で表示（特別扱いしない）
- **選択中カテゴリが空になる**（BP全削除等の再配信）: 先頭カテゴリへフォールバック

## 7. 自己反証

- 「順序を最小sortPriorityから導出」への反例: サブカテゴリ間で意図しない順になる場合がある（例: チェスト200が物流先頭になり物流カテゴリがサイドバー2番目に来る）。→ 順序が気に入らない場合の調整手段は**マスタのsortPriority側を直す**こと（master-refineの再計算と同じ思想）。UI側に順序テーブルを持たない
- 「検索中サイドバー無効化」への反例: 検索しながらカテゴリを絞りたいユースケース。→ 73種の規模では横断検索で十分（YAGNI）。必要になったら様式更新から入る
- 「category自由文字列」への反例: タイポで新カテゴリが生えて気づかない。→ 検証計画5の実マスタ目視で検出する。foreignKey化（カテゴリマスタ新設）は導出可能テストにより不採用
