# クリーンルーム（空気純度）システム 設計書

- 日付: 2026-06-05
- 対象: moorestech V8 mod（半導体／ICチップ生産）
- ステータス: 設計承認済み（数値バランスは実装プランで確定）

## 1. 概要

半導体（EUV露光〜ICチップ）生産に「空気純度」の概念を導入する。プレイヤーは壁で密閉したクリーンルームを建て、空気清浄機とフィルターで不純物濃度を維持する。部屋の清浄度（クラス）が、製造の基礎歩留まりと製造可能なチップレベルの上限を決める。

設計の核となる決定:

- クリーンルームは **壁で完全に囲った3D密閉空間** として検出する。
- 純度は **不純物濃度（個/m³）** で表し、閾値で **クラス** が決まる。
- クラスが **基礎歩留まり率の床** と **最大チップレベルの天井** を規定する。
- 不純物は複数の汚染源から増え、空気清浄機（電力＋フィルター消費）で除去する。
- アイテムを部屋外へ取り出しても、そのアイテムは変化しない（純度は製造の瞬間にのみ参照される、部屋の状態属性）。

## 2. 部屋検出（3D密閉箱）

- **クリーンルームブロック（壁）** で完全に囲まれた立体空間を1つの部屋として検出する。
- 内部の空セルから3D flood-fill を行い、フィルが外部へ漏れる／規定の上限体積を超えた場合は **密閉していない＝部屋不成立** とする。
- 境界を貫通してよいのは **ドア／アイテムハッチ／パイプコネクタ** のみ。それ以外の壁の欠け（隙間）は即リークとして扱う。
- ブロックの設置・破壊のたびに、影響範囲の部屋を再検出する。
- 各部屋は不純物状態（不純物総数 N、体積 V）を保持し、セーブ／ロード対象とする。

### 実装コスト（重い箇所・実装プランで詳細化）

- 3D flood-fill による密閉判定と、設置/破壊時の影響範囲の再検出。
- 部屋の動的な結合／分割（壁を足す・壊すで部屋が繋がる・割れる）。
- 部屋ごとの不純物状態の永続化（セーブ／ロード）。

## 3. 純度モデル

各部屋は **不純物濃度 C = 不純物総数 N ÷ 部屋体積 V（個/m³）** を持つ。

- 汚染源は毎秒 **A 個** を加算する（A は §5 の各汚染源の合計。機械数・ハッチ数などに比例）。
- **空気清浄機は「毎秒一定体積 q(m³/s) の空気を処理する」**。除去量 = `n台 × q × C(濃度)`。固定個数除去ではない。
- これにより不純物総数の時間変化は `dN/dt = A − n·q·C = A − n·q·(N/V)`。
- **平衡濃度 `C_eq = A / (n·q)`** に安定収束する。

### なぜ「体積処理式」か（固定個数除去を採用しない理由）

固定個数除去（除去 = n×R、濃度に依存しない定数）にすると、`dN/dt = A − n·R` が定数になり、
N は 0 にクランプ（常に最良クラス）か、無限大に発散（常に最悪クラス）の二択に崩壊する。
安定した内部値が存在せず、多段クラスが機能しない。体積処理式なら平衡濃度に収束し、クラスが連続スペクトルになる。

### 体積 V の役割 ＝ バースト緩衝

平衡濃度 C_eq は体積に依存しない（V は式から消える）。体積が効くのは **過渡（バースト）** に対して:

- ドア開閉やハッチ搬入出は固定個数の不純物を一度に投入する。
- 大部屋はこのスパイクを薄めて吸収する（許容的だが、平衡へ戻る回復は遅い）。
- 小部屋はスパイクで濃度が大きく跳ねる。

これにより3D体積判定が「大部屋＝バースト耐性が高い」という形で意味を持つ。

## 4. クラスと効果（既存歩留まりレバーとの役割分担）

濃度 C の閾値で **クリーンルームクラス** が決まり、クラスが2つを規定する。

| レバー | 何を決めるか |
|---|---|
| **クリーンルームクラス（純度）** | 基礎歩留まり率の **床** ＋ 製造可能な **最大チップレベルの天井** |
| **品質モジュール** | 天井の範囲内での出力レベルの **分布**（上位レベルが出る確率） |
| **EUV 20%失敗 / 歩留まり改善モジュール** | レシピ単位の **失敗率** |

純度＝「天井と床」、モジュール＝「その中の調整」と役割を分離し、多重の歩留まり係数が混ざって出力が読めなくなる事態を避ける。

## 5. 汚染源（A に合算）

以下4つすべてが不純物を増やす:

- **常時の自然増加**: ベースで微増。清浄機を常時稼働させて相殺する前提（フィルター消費のベース負荷）。
- **製造機の稼働**: EUV 等の稼働中に不純物を排出。機械を詰めるほど汚染が増える。
- **ドアの開閉（人の出入り）**: プレイヤー通過で大きめに加算。自動化を促す。
- **ハッチのアイテム搬入出**: 搬入出ごとに加算。搬入出が多いほど汚れる＝集約の根拠。

## 6. 空気清浄機 ＋ フィルター

- 常時 **電力** を消費する（ElectricMachine 系）。
- **フィルターは仕事量ベースで消費する** ＝ 除去した不純物量に比例。
  - 平衡時のフィルター消費は総汚染レート A に比例する。
  - → 汚い部屋ほどフィルターを食う。ハッチ／ドアを減らす動機が直接生まれる。
- 複数台で除去能力 `n·q` が加算される。機械を詰めるほど清浄機も増設が必要になる。

## 7. I/O の役割分担

- **アイテムハッチ**: アイテムの搬入出。低スループット（集約の根拠）かつ汚染源。
- **パイプコネクタ**: 流体の搬入出（超純水・IPA 等）。
- **ドア**: プレイヤーの出入り。汚染が大きく、自動化を促す。

## 8. 製造機とクリーンルームの関係

- 半導体製造機（EUV 露光式半導体製造装置 等）は **有効なクリーンルーム内でのみ稼働** する。
- 部屋が無効化された場合（壁破壊・リークで密閉が崩れる、または機械が部屋外にある）、機械は **外に置いたのと同じ状態＝単純に稼働を停止** する。
- 部屋のクラスが製造対象チップの要求に満たない場合は、最大チップレベルの天井で出力を制限する（稼働自体は継続）。

## 9. アイテム外部取り出し時の挙動

- アイテムをクリーンルーム外へ取り出しても、そのアイテムは変化しない。
- 純度はアイテムが持つ状態ではなく、製造が行われる瞬間に参照される部屋の属性として扱う（アイテム単位の汚染度トラッキングは行わない）。

## 11. 実装プランで確定する数値・詳細パラメータ

- クラスの濃度閾値（クラス境界となる個/m³）と、各クラスの基礎歩留まり率・最大チップレベル。

- 部屋状態のセーブ／ロード形式。
