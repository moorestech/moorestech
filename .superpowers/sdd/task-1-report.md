## 生成型（downstreamタスク向け・実測）

- 名前空間: `Mooresmaster.Model.ConnectToolsModule` / ローダー `Mooresmaster.Loader.ConnectToolsModule.ConnectToolsLoader`
- ルート型 `ConnectTools`（`.Data` が `ConnectToolMasterElement[]`）
- `ConnectToolMasterElement`: `ConnectToolGuid`(Guid), `Name`(string), `ToolType`, `ImagePath`(string),
  `SortPriority`(int), `InitialUnlocked`(bool), `LengthPerUnit`(double), `RequiredItems`(ConnectToolRequiredItemElement[])
- **toolTypeは生成enum**（`string`ではない）。既存enum（RecipeViewType）と同じく nested const 型として生成される:
  `ConnectToolMasterElement.ToolTypeConst.ElectricWire / .Rail / .GearChain`。
  brief記載の`string ToolType`は不正確 — downstreamのtoolType分岐はこのenumで行うこと。
- `ConnectToolRequiredItemElement`: `ItemGuid`(Guid), `Count`(int)

## 懸念 / 後続タスクへの申し送り

1. **実マスタ(../moorestech_master)未対応（重要）**: MasterHolderはconnectToolsをmandatoryロードするため、
   `../moorestech_master/.../master/`にconnectTools.jsonが無い状態で実プレイ起動すると
   KeyNotFound/MooresmasterLoaderExceptionでboot失敗する。spec §1の../moorestech_master編集（items削除等）を行う
   後続タスクで**必ずconnectTools.json（実データ3エントリ・銅のワイヤー/補強棒材+鉄板/鉄のワイヤー）を同時追加**すること。
   本タスクはbrief指定通りテストJSONのみ対応（実プレイ検証は未実施）。
2. **requiredItems要素名**: 既存`ConstructionRequiredItemElement`（BlocksModule）を再利用せず、
   ConnectToolsModule独自の`ConnectToolRequiredItemElement`を生成。消費ロジック実装タスクはこの型を使用のこと。
3. **テストデータのinitialUnlockedは全てfalse**: テストmodに初期解放済みconnectToolは無い。
   default選択やビルドメニュー表示の解放依存テストを追加する際は、テストmod側で必要に応じてtrueへ調整すること。
4. **imagePathは必須**: 実データ追加時（後続タスク）は3エントリ全てにアイコンパスを指定すること。
