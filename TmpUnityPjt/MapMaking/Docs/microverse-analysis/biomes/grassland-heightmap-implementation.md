# 草原バイオーム ハイトマップ実装方針

> **本ドキュメントは未検証の実装方針であり、実装時にはプロトタイプ検証を経て修正される前提です。この方針に固執する必要はありません。**

## 1. 使用スタンプの視覚分析

### Highlands 03 スタンプ

- **アセットパス**: `Assets/Rowlan/Terrain/Stamps/Highlands/Heightmaps/Highlands 03.png`
- **解像度**: 4096 x 4096 (R16)
- **MicroVerse での適用**: Override モード、Y スケール 0.10（高さ振幅 約12.5m）

#### 視覚的特徴

スタンプ画像を確認すると、以下の特徴が読み取れる：

1. **大きな明暗のうねり**: 画面全体に横方向（水平方向）に伸びる幅広い明暗の帯が走っている。これは低周波の丘陵パターンに対応する
2. **丸みを帯びたローブ構造**: 明部（高い部分）は丸い塊状で、鋭い稜線やピークがない。Highlands（高地）という名前だが、角が取れたなだらかな丘の集合体
3. **中間調が支配的**: 極端な白（最高点）や黒（最低点）は少なく、画像全体がグレーの中間帯に集中している。ヒストグラムが狭い = 高低差が小さい地形に向いている
4. **異方性**: 水平方向に細長いパターンが目立ち、Z方向（縦方向）よりX方向（横方向）の変化波長が長い。MicroVerse では 271度回転で適用されているため、ワールド空間では Z 方向に長い丘陵パターンとなる
5. **暗い谷状の溝**: 数カ所に暗い（低い）領域が点在するが、鋭い渓谷ではなく緩やかなくぼみ

#### Y=0.10 の意味

本来は起伏の大きい高地テクスチャを Y 方向に 10% に圧縮している。4096px テクスチャが持つすべての微細な高低差が 12.5m の範囲に押し込められ、結果として「ほぼ平坦だが完全ではない」という草原らしい起伏になる。


## 2. 地形特徴の分解

Highlands 03 を Y=0.10 で適用した草原地形は、以下の要素に分解できる：

### 要素A: 広域うねり（支配的）

- **波長**: 200-500m 程度
- **振幅**: 5-10m（全高さ変動の大部分）
- **形状**: 正弦波的ではなく、丸みを帯びた不規則なうねり
- **方向性**: やや異方的（Z方向に長い）

### 要素B: 中間スケールの凹凸

- **波長**: 50-150m 程度
- **振幅**: 2-4m
- **形状**: 広域うねりの上に乗る二次的な起伏
- **役割**: 完全な単調さを防ぎ、地形に自然な複雑さを加える

### 要素C: 微細な起伏（最小限）

- **波長**: 10-30m 程度
- **振幅**: 0.5-1.5m
- **特徴**: テクスチャ由来の細かいうねり。Y=0.10 の圧縮によりほぼ目立たない
- **役割**: 完全にフラットな面を回避し、テクスチャタイリングが目立つのを防ぐ

### 欠如する要素

- **鋭い崖・急斜面**: 一切存在しない
- **明確なピーク**: 山頂のような尖った構造がない
- **渓谷・溝**: 深い切れ込みがない
- **テラス構造**: 段差状の地形がない

これが「開けた草原の広がり」という印象を生む核心であり、他のバイオームとの最大の差別化要因。


## 3. プロシージャル再現アルゴリズム

### 方針: 現行 GrasslandBiome の簡素化

現行の `GrasslandBiome.SampleHeight()` は Broken Lands 式のドメインワープ + 渓谷カービング + プラトー平坦化 + リッジ加算という複雑なパイプラインを持つ。これはフォールバックバイオームとして全領域をカバーするための汎用設計であり、MicroVerse の草原スタンプが示す「シンプルになだらかな丘陵」とは性格が異なる。

草原の HeightMap 再現に特化する場合、以下の簡素なアプローチで十分と考えられる：

### 擬似コード

```
function SampleGrasslandHeight(worldX, worldZ, offsets):
    // === ステップ1: 低周波 fBm で広域うねり ===
    // 2-3オクターブの少ないオクターブ数でディテールを抑える
    // persistence=0.3-0.4 の低い値で高周波成分をさらに抑制
    terrain = SampleFBm(worldX, worldZ,
        frequency=0.0004,      // 非常に低い周波数 → 波長 ~2500m
        offsets, offsetStart=0,
        persistence=0.35,      // 高周波成分を大きく減衰
        lacunarity=2.2,
        octaves=3)             // 少ないオクターブ

    // === ステップ2: 軽度のドメインワープ（任意） ===
    // 正弦波的な規則性を崩すために微弱なワープを適用
    // ワープなしでも成立するが、あると自然さが増す
    warpX = SampleFBmRaw(worldX, worldZ,
        frequency=0.0003, offsets, warpXStart,
        persistence=0.4, lacunarity=2.0, octaves=2) * warpStrength
    warpZ = SampleFBmRaw(worldX, worldZ,
        frequency=0.0003, offsets, warpZStart,
        persistence=0.4, lacunarity=2.0, octaves=2) * warpStrength

    terrain = SampleFBm(worldX + warpX, worldZ + warpZ, ...)

    // === ステップ3: 丸み付与 ===
    // smoothstep で中間調への集中を再現（明暗の極端さを抑える）
    terrain = smoothstep(terrain)  // t*t*(3-2t)

    // === ステップ4: スケーリング ===
    return baseHeight + terrain * hillAmplitude
```

### 既存 NoiseSampler との対応

| 擬似コード関数 | 既存実装 |
|---------------|---------|
| `SampleFBm` | `NoiseSampler.SampleFBm()` — そのまま使用可能 |
| `SampleFBmRaw` | `NoiseSampler.SampleFBmRaw()` — ドメインワープ用 |
| `smoothstep` | `t * t * (3 - 2 * t)` — インライン実装 |

新しいノイズ関数の追加は不要。既存の `NoiseSampler` で完結する。


## 4. パラメータ提案

### GrasslandBiomeConfig に必要なパラメータ

MicroVerse のスタンプ分析結果を再現するためのパラメータ提案：

| パラメータ | 提案値 | 根拠 |
|-----------|-------|------|
| `baseHeight` | 0.03 | 現行値を維持。海面(0.01)よりやや上 |
| `hillAmplitude` | 0.02 | MicroVerse の 12.5m / terrainHeight(600m) ≈ 0.021。現行 0.1 は大きすぎる |
| `frequency` | 0.0004 | 波長 ~2500m。スタンプの広域うねりに対応 |
| `octaves` | 3 | 高周波成分を抑え、なだらかさを維持 |
| `persistence` | 0.35 | 2,3オクターブ目の寄与を強く減衰 |
| `lacunarity` | 2.2 | 標準的な値 |
| `domainWarpStrength` | 100-200 | 現行 750 は強すぎる。微弱なワープで十分 |
| `domainWarpIterations` | 1 | 1回で十分。0でもよい |

### 不要にできるパラメータ（草原の場合）

現行の `GrasslandBiome` は汎用フォールバックとして多くの機能を持つが、MicroVerse の草原を忠実に再現する場合、以下は不要：

| パラメータ | 現行値 | 理由 |
|-----------|-------|------|
| `canyonDepth` | 0 | スタンプに渓谷構造がない。現行も 0 なので問題なし |
| `ridgeBlend` | 0.8 | スタンプに稜線構造がない。0 にすべき |
| `plateauFlatten` | 0.7 | プラトー平坦化はスタンプの特徴にない。ただし smoothstep を別途使う場合は代替手段として検討 |
| `exponent` | 2.0 | コントラスト強調はスタンプの中間調集中に反する。1.0 にすべき |

### 振幅に関する重要な考察

MicroVerse の草原は高さ振幅 12.5m だが、これは MicroVerse のワールド全体の高低差に対する比率で「非常に低い」とされている。MapGenerator では `terrainHeight=600m` に対して `hillAmplitude=0.02` で `600 * 0.02 = 12m` となり、近い値になる。

ただし、MapGenerator のバイオーム補間システムでは隣接バイオームの高さが影響するため、実効的な草原部分の高低差は `hillAmplitude` 単体では決まらない。`biomeBlendRadius` による平滑化も考慮が必要。


## 5. 既存コードとの差分

### 現行 GrasslandBiome.SampleHeight() の処理フロー

```
WarpCoords (5オクターブ x2、750m強度、2回反復)
  → メインFBm (9オクターブ)
    → 渓谷カービング (6オクターブ)
      → プラトー平坦化 (smoothstep)
        → べき乗コントラスト (exponent=2.0)
          → 稜線加算 (7オクターブ)
```

### 提案する草原特化フロー

```
軽度ワープ (2オクターブ x2、100-200m強度、1回反復)
  → 低周波FBm (3オクターブ)
    → 丸み付与 (smoothstep)
```

### 主な差分

| 項目 | 現行 | 提案 | 差分の理由 |
|------|-----|------|-----------|
| ノイズオフセット数 | 29個 | 7個 | オクターブ削減による |
| ドメインワープ | 750m, 2反復 | 100-200m, 1反復 | 草原にうねりのような有機的変形は不要 |
| メインFBm | 9オクターブ | 3オクターブ | 高周波ディテールが草原のなだらかさに反する |
| 渓谷カービング | 使用可能（depth=0） | 不要 | スタンプに渓谷なし |
| 稜線加算 | blend=0.8 | 不要 | スタンプに稜線なし |
| べき乗 | 2.0 | 1.0（線形） | 中間調集中にはコントラスト強調不要 |

### 実装アプローチの選択肢

**A. 既存 Config のパラメータ調整のみ**

現行コードを変更せず、パラメータだけで近づける方法：
- `canyonDepth=0`, `ridgeBlend=0`, `plateauFlatten=0`, `exponent=1.0` で不要機能を無効化
- `octaves=3`, `persistence=0.35`, `frequency=0.0004` で低周波に変更
- `domainWarpStrength=150`, `domainWarpIterations=1` でワープを弱める
- `hillAmplitude=0.02` で振幅を下げる

長所: コード変更なし。短所: 不要な分岐が残るが、`canyonDepth=0` 等の条件分岐で実行時コストはほぼゼロ。

**B. 草原専用 SampleHeight を新設**

`GrasslandBiome` のフォールバック役割はそのままに、SampleHeight 内部のロジックを草原特化にリファクタリングする方法。

長所: コードが意図を明確に表す。短所: Grassland がフォールバック（他のどのバイオームにも分類されない領域）を担っている設計上、過度に草原特化すると他バイオームが無効化された場合の見栄えに影響する。

**推奨: アプローチ A**。現行のパイプラインは十分汎用的であり、パラメータ調整だけで草原のなだらかさを実現できる。フォールバックとしての柔軟性も失わない。


## 6. 未検証事項

以下の項目は実装・プロトタイプ検証を経て確認する必要がある：

1. **振幅 0.02 での視覚的インパクト**: 12m の高低差がゲームプレイ上十分な空間変化を与えるか。MicroVerse では 4096px スタンプの解像度がディテールを補っていたが、3オクターブの fBm で同等の視覚品質が得られるかは未確認

2. **バイオーム境界での補間**: `hillAmplitude=0.02` (草原) と隣接バイオーム（例: Alpine の `amplitude=0.55`）の差が大きい場合、`biomeBlendRadius=40` での補間が自然に見えるか

3. **異方性の必要性**: MicroVerse のスタンプは 271度回転で Z 方向に長い丘陵パターンを作っていた。等方性のPerlinノイズで再現できるか、あるいは周波数を X/Z で異ならせる必要があるか

4. **ドメインワープの必要性**: ワープなし（`domainWarpIterations=0`）でも十分自然な草原になる可能性がある。プロトタイプで比較検証が必要

5. **フォールバック役割との両立**: パラメータをなだらか方向に振ると、他バイオームが全て無効の場合にマップ全体が平坦すぎる可能性がある。フォールバックとして許容できるかの確認

6. **smoothstep の効果**: テクスチャの中間調集中を再現するために `plateauFlatten` の smoothstep を流用する方法が有効か、それとも素の fBm 出力で十分か

7. **パフォーマンス影響**: 3オクターブ化によるパフォーマンス改善の実測値。現行 29 オフセット → 7 オフセットの削減が全体生成時間に与える影響
