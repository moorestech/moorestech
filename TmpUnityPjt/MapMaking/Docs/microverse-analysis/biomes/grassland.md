# 草原エリア（Grassland）バイオーム分析

## 1. バイオーム概要

草原エリアは、木が一切存在しない開けた草原バイオームである。テクスチャ構成は全バイオーム中最もシンプル（1レイヤーのみ）だが、ディテール（草花）構成は全バイオーム中最もリッチ（12スタンプ）という対照的な設計を持つ。視覚的な豊かさはテクスチャではなく、多種多様な草のディテールによって実現されている。

| 項目 | 値 |
|------|------|
| ルートオブジェクト | `MicroVerse/草原エリア` |
| ワールド座標 | (449.43, 78.00, 298.36) |
| ルートスケール | (1.46, 1.00, 1.46) |
| SplineArea bounds | Center: (346.15, 0.00, 207.22), Extents: (568.41, 100000.00, 378.03) |
| HeightStamp 数 | 1 |
| TextureStamp 数 | 1（全バイオーム最少） |
| DetailStamp 数 | 12（全バイオーム最多） |
| TreeStamp 数 | 0（唯一の木なしバイオーム） |
| ObjectStamp 数 | 1 |

### 階層構造

```
草原エリア [SplineContainer, SplineArea]
  └─ Dry Cliffs [FalloffOverride]
       ├─ Clear Stamp [ClearStamp]
       ├─ Rubble/ [ObjectStamp + DetailStamp x7]
       ├─ Grass (Medium)/ [DetailStamp x5]
       ├─ Grass 01/ [TextureStamp "Ground"]
       └─ Highlands 03 (Height Stamp) [HeightStamp]
```

### 他バイオームとの構成比較

| バイオーム | Texture | Detail | Tree | Object |
|-----------|---------|--------|------|--------|
| **草原** | **1** | **12** | **0** | **1** |
| 砂漠 | 2 | 1 | 1 | 2 |
| メサ | 4 | 20 | 5 | 8 |
| サバンナ | 7 | 3 | 2 | 0 |
| 森林 | 2 | 12 | 2 | 1 |
| 林 | 3 | 6 | 3 | 0 |

草原はテクスチャ最少・ツリーゼロという極端な構成。ディテール数は森林と並び最多タイだが、森林は木と倒木による視覚的要素があるのに対し、草原は草花だけで景観を成立させる必要があるため、ディテールの多様性がより重要になる。


## 2. デザイン意図

草原エリアのデザインコンセプトは「見た目の空虚さ」と「近づいたときの草の豊かさ」の二面性である。

**遠景**: なだらかな丘陵に単一のテクスチャが敷かれ、木もオブジェクトもほぼない。開放感と空虚さを演出する。HeightStamp の Y スケール 0.10 によるわずかな起伏のみが地形的な変化を生む。

**近景**: 12種類の DetailStamp が 2 つのノイズグループ（Shurubu 系 / Grass 系）で配置され、足元に豊かな草のバリエーションを展開する。マルチスケールのノイズにより、均一にならない自然な群生パターンが生まれる。

**視覚的アンカー**: 木がないため、地面レベルの RubbleDense/Sparse（岩の散乱物）が唯一の立体的なアンカーとなる。これにより完全な平坦感を回避しつつ、草原の開放感を損なわない。

この設計は「テクスチャの複雑さ = 視覚品質」ではないことを示す好例であり、DetailStamp の多様性がバイオームの説得力を決定するケースである。


## 3. HeightMap 構成

草原エリアには HeightStamp が **1つだけ** 配置されている。

### Highlands 03 (Height Stamp)

| パラメータ | 値 |
|-----------|------|
| テクスチャ名 | Highlands 03 |
| アセットパス | `Assets/Rowlan/Terrain/Stamps/Highlands/Heightmaps/Highlands 03.png` |
| 解像度 | 4096 x 4096 (R16) |
| ワールド座標 | (325.55, 84.24, 183.20) |
| 回転 | Y軸 271.24度（ほぼ270度、X/Z軸が入れ替わる） |
| lossyScale | (512.23, **12.46**, 1487.07) |

#### HeightStamp パラメータ

| パラメータ | 値 | 意味 |
|-----------|------|------|
| mode | **Override** | 既存地形を完全に上書き |
| blend | 1 | 完全適用 |
| power | 1 | リニアマッピング |
| invert | False | 反転なし |
| twist | 0 | UV歪みなし |
| erosion | 0 | 侵食エフェクトなし |
| useHeightRemap | False | リマップカーブ不使用 |
| scaleOffset | (1.0, 1.0, 0.0, 0.0) | タイリング/オフセットなし |

#### Falloff 設定

| パラメータ | 値 |
|-----------|------|
| filterType | Box |
| falloffRange | (0.80, 1.00) |
| easing | Linear |
| noise | None |

スタンプ範囲の 80% までは完全適用され、外側 20% で Linear にフェードアウトする。

#### ワールド空間での有効範囲

271度回転（ほぼ270度）によりローカルの X/Z が入れ替わる。

| 軸 | 範囲 | 幅 |
|----|------|-----|
| X | 69 ~ 582 | 約512m |
| Z | -560 ~ 927 | 約1487m |
| Y (高さ) | 中心84.24, 振幅 +/-6.23 | **約12.5m** |

#### 地形特性の要点

**Y スケール 0.10 が草原らしさの鍵。** 本来は起伏の大きい高地テクスチャ（Highlands 03）を、Y 方向に 10% まで圧縮することで、なだらかな丘陵地形に変換している。4096x4096 の高解像度テクスチャが持つ微細な高低差パターンが、12.5m の範囲に凝縮されることで、肉眼では「ほぼ平坦だが完全ではない」という自然な草原の起伏を生む。

### FalloffOverride ("Dry Cliffs")

SplineArea 配下の全スタンプに適用される、バイオーム境界のフェードアウト設定。

| パラメータ | 値 |
|-----------|------|
| filterType | SplineArea |
| splineAreaFalloff | 30 |
| splineAreaFalloffBoost | 10 |
| easing | EaseInOut |
| noise | None |

スプライン境界から 30m の幅で EaseInOut カーブにより滑らかに減衰する。Boost=10 により境界付近でのフェードが強調される。

### ClearStamp

| パラメータ | 値 |
|-----------|------|
| lossyScale | (436.62, 120.00, 436.62) |
| clearTrees | True |
| clearDetails | True |
| clearObjects | True |

草原エリア内の既存植生を全てクリアし、このバイオーム固有の構成で再配置するための下地を作る。

### SplineArea 設定

| パラメータ | 値 |
|-----------|------|
| ノット数 | 24 (closed=True) |
| closedMode | Area |
| sdfRes | k1024 |
| maxSDF | 150 |

24ノットの閉じたスプラインで不規則な多角形を描く。西側（ノット6-10）が大きく広がり、東側（ノット11-17）は比較的直線的な形状。


## 4. テクスチャ構成

草原エリアのテクスチャ構成は全バイオーム中最もシンプルで、**1レイヤーのみ**である。

### TextureStamp 一覧

| # | グループ | レイヤー | weight | ノイズ |
|---|---------|---------|--------|--------|
| 1 | Ground | Grass | 1.0 | None |

**ノイズなし、フィルタなし、weight=1 で無条件に Grass テクスチャを全面に適用する。** 斜面フィルタや高度フィルタも設定されておらず、スプラインエリア内の全領域が単一テクスチャで覆われる。

### 他バイオームとの比較

- サバンナ: 7 レイヤー（GrassGreen / GrassDark / GrassGreenDirt を WormFBM で混在）
- メサ: 4 レイヤー（SandLarge / SandFine / 砂利 / 瓦礫を斜面条件やノイズで切り替え）
- 森林: 2 レイヤー（Grass + SoilPine を WormFBM で混在）
- **草原: 1 レイヤー（Grass のみ）**

草原がこれほど単純なテクスチャ構成でも視覚的に成立するのは、地面テクスチャが草の「下地」にすぎないからである。12種類の DetailStamp が草のカバレッジを稠密に配置するため、地面テクスチャは草の隙間からわずかに見える程度に過ぎない。


## 5. 草花・ディテール構成

草原エリアの最大の特徴であり、12 個の DetailStamp が 2 つのノイズグループに分かれて配置されている。

### DetailStamp 一覧

#### Shurubu グループ（低周波ノイズ: freq=3.98, amp=2.62）

| # | プロトタイプ | ノイズタイプ | freq | amp |
|---|-------------|-----------|------|-----|
| 1 | Shurubu_1 | None | 3.98 | 2.62 |
| 2 | Shurubu_2 | None | 3.98 | 2.62 |
| 3 | Shurubu_3 | None | 3.98 | 2.62 |
| 4 | Shurubu_4 | None | 3.98 | 2.62 |
| 5 | Shurubu_5 | None | 3.98 | 2.62 |

#### Grass グループ（高周波ノイズ: freq=10, amp=1）

| # | プロトタイプ | ノイズタイプ | freq | amp |
|---|-------------|-----------|------|-----|
| 6 | Grass_0 | None | 10 | 1 |
| 7 | Grass_1 | None | 10 | 1 |
| 8 | Grass_2 | None | 10 | 1 |
| 9 | Grass_3 | None | 10 | 1 |
| 10 | Grass_4 | None | 10 | 1 |
| 11 | Sorrel | None | 10 | 1 |
| 12 | (Grass系の追加) | None | 10 | 1 |

### 2つのノイズグループによるマルチスケール草配置

この構成の核心は **2つの異なる周波数のノイズグループによるマルチスケール変化** にある。

**Shurubu グループ（freq=3.98, amp=2.62）**: 低周波・高振幅のノイズ。大きなスケール（数十メートル単位）で草の群生パターンを形成する。振幅 2.62 は比較的大きく、密な群生と疎な区域がはっきり分かれる。5 種の Shurubu バリエーションにより、同じ群生パターン内でも見た目の多様性が確保される。

**Grass グループ（freq=10, amp=1）**: 高周波・低振幅のノイズ。小さなスケール（数メートル単位）で草の密度を細かく変化させる。振幅 1.0 は Shurubu より控えめで、均一すぎず偏りすぎない中程度の変化を生む。Grass_0-4 の 5 種 + Sorrel（スイバ）で 6 種のバリエーション。

この 2 グループの重ね合わせにより:
- 遠景では Shurubu の低周波パターンが見え、大きな群生の塊として認識される
- 近景では Grass の高周波パターンが加わり、細かい密度変化と種類の多様性が見える
- 合計 12 種のプロトタイプが混在するため、どこを見ても均一なタイリング感がない

### DetailStamp システムとの対応

DetailStamp の密度マップ生成は GPU シェーダー（`DetailFilter.shader`）で行われる。各 DetailStamp の密度は以下の式で決まる:

```
密度 = DoFilters(uv) * sdf * mask * texMask * _Density
```

同一プロトタイプに対する複数スタンプの結果は **max 合成**（加算ではなく最大値）でマージされる。草原エリアでは各プロトタイプが 1 スタンプずつ割り当てられているため、マージ処理は発生しない。


## 6. 樹木構成

**草原エリアには TreeStamp が存在しない。** これは全 7 バイオーム中、草原のみが持つ特徴である。

| バイオーム | 樹木スタンプ数 | 主要な木 |
|-----------|-------------|---------|
| 砂漠 | 1 | Olivetree, Olivebush |
| メサ | 5 | Senita, Saguaro, Cactus, Opuntia |
| サバンナ | 2 | Acacia, Bush |
| 森林 | 2 | RedPine（サイズ違い2種） |
| 林 | 3 | Spruce, Fir, Stone |
| 岩石山 | 2 | Olivetree, Olivebush |
| **草原** | **0** | **なし** |

木がないことは、草原バイオームにおいて以下の設計上の帰結をもたらす:

1. **ディテールが視覚的主役**: 他バイオームでは木が遠景のシルエットを形成するが、草原では草のディテール密度が唯一の視覚的情報源。これが 12 スタンプという豊富なディテール構成の理由
2. **SDF オクルージョンが不要**: TreeStamp からの SDF 距離マスクがないため、DetailStamp は木の根元を避ける制御が不要。配置ロジックがシンプルになる
3. **パフォーマンス余力**: 木のレンダリングコストがゼロのため、その分をディテール密度に振り向けられる


## 7. オブジェクト構成

草原エリアの ObjectStamp は **1つだけ** で、Rubble（瓦礫・散乱岩）のみが配置される。

### Rubble

| パラメータ | 値 |
|-----------|------|
| density | 3.02 |
| プロトタイプ数 | 6 |
| プロトタイプ | RubbleDense_1-3, RubbleSparse_1-3 |
| slopeAlignment | 0（地形法線への整列なし） |
| ノイズ | WormFBM |

#### プロトタイプ詳細

- **RubbleDense_1-3**: 密集した小石・岩片の塊。比較的まとまったシルエット
- **RubbleSparse_1-3**: まばらに散らばった岩片。より広い範囲に薄く分布するシルエット

6 種のバリエーションが重み付きランダムで選択される。`slopeAlignment=0` のため、オブジェクトは地形に沿わず水平に配置される（小さな瓦礫なので問題にならない）。

#### WormFBM ノイズによる配置パターン

WormFBM はワーム（虫食い）状の不規則なパターンを生成するノイズで、配置密度に空間的な変化を与える。瓦礫が均一に散らばるのではなく、帯状や塊状の偏りを持って配置される。

#### 他バイオームのオブジェクト構成との比較

- メサ: 8 スタンプ（Strate, BigMesa, ThinMesa, Boulders, RubbleDense/Sparse）
- 砂漠: 2 スタンプ（High Desert Cliffs, Dense Rubble）
- 岩石山: 3 スタンプ（Desert Cliffs x2, Desert Rocks）
- **草原: 1 スタンプ（Rubble のみ）**

草原のオブジェクトは、木がない空間に最低限の地面レベルの視覚的アンカーを提供する役割。岩壁や大きな岩塊ではなく、あくまで散乱した小石程度に留めることで草原の開放感を維持している。


## 8. MapGenerator 再現パラメータ

MicroVerse の草原エリアを MapGenerator で再現するための設計要件をまとめる。

### 地形生成

| 項目 | MicroVerse 値 | MapGenerator 方針 |
|------|-------------|-----------------|
| 高さの振幅 | ~12.5m (Y scale 0.10) | fBm の振幅を全体高さの 1-2% に抑える |
| ノイズの周波数 | 低周波（4096px テクスチャの緩やかなパターン） | 1-2 オクターブの低周波 fBm |
| ブレンドモード | Override | バイオーム weight=1.0、境界でのみブレンド |
| フォールオフ | SplineArea + EaseInOut, 30m幅 | SDF 距離関数 + EaseInOut カーブ |
| 異方性 | Z方向が X の約3倍（271度回転による） | X/Z で異なるノイズスケール |

### テクスチャ配置

| 項目 | MicroVerse 値 | MapGenerator 方針 |
|------|-------------|-----------------|
| レイヤー数 | 1（Grass のみ） | Grass テクスチャを無条件適用 |
| フィルタ条件 | なし | バイオームマスクのみで切り替え |

### ディテール配置

| グループ | freq | amp | 対応するノイズ設計 |
|---------|------|-----|-----------------|
| Shurubu (5種) | 3.98 | 2.62 | 低周波 Perlin, 高振幅。大スケールの群生パターン |
| Grass (7種) | 10 | 1.0 | 高周波 Perlin, 中振幅。小スケールの密度変化 |

**マルチスケール草配置の再現手順**:

1. 低周波ノイズ（freq~4）で大きな群生領域を定義し、Shurubu 系プロトタイプを配置
2. 高周波ノイズ（freq~10）で細かい密度変化を加え、Grass 系プロトタイプを配置
3. 各グループ内の複数プロトタイプは等確率で選択（ランダム分散）
4. 2グループの重ね合わせで、どの距離でも均一感のない自然な草原を実現

### オブジェクト配置

| 項目 | MicroVerse 値 | MapGenerator 方針 |
|------|-------------|-----------------|
| 種類 | RubbleDense/Sparse のみ | 小石・瓦礫プレハブ 6 種 |
| density | 3.02 | 同程度の密度 |
| ノイズ | WormFBM | Perlin ベースの不規則パターンで代替可 |
| slopeAlignment | 0 | 水平配置 |

### 樹木

**配置しない。** 草原バイオームの定義として木は 0 本。

### バイオーム境界

- フォールオフ幅: 30m
- イージング: EaseInOut
- 草原側の weight を 1.0 に設定し、境界の 30m 幅でのみ隣接バイオームとブレンド
- 境界内では DetailStamp の密度もフォールオフに比例して減少させる


## 地形生成パイプライン（処理順）

```
1. ClearStamp     → 既存植生・オブジェクトをクリア
2. HeightStamp    → Highlands 03 テクスチャで地形を Override
3. FalloffOverride → SplineArea 境界で EaseInOut フェードアウト (30m幅, boost=10)
4. TextureStamp   → Grass テクスチャレイヤーを全面配置
5. ObjectStamp    → Rubble (瓦礫) オブジェクトを WormFBM ノイズで散布
6. DetailStamp    → 12種の草ディテールを 2 ノイズグループで配置
```
