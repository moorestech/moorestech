# 砂漠エリア（Desert）総合バイオーム分析

## 1. バイオーム概要

砂漠エリアは BK PureNature の "High Cliffs with Trees"（Oasis プリセット）を基盤とした、**乾燥した峡谷と崖が支配する荒涼地帯**である。ワールド座標 (1412.95, 146.29, 808.38) に位置し、東西約750m x 南北約1200mの広大な不定形領域をスプラインで囲む。

視覚的な特徴は以下の3点に集約される：

1. **砂一色のベース地表** --- SandFine テクスチャが全面を覆い、わずかな泥パッチがワームFBMノイズで有機的に点在する。色彩の単調さが「乾燥」を直接的に伝える
2. **峡谷+崖の二重地形構造** --- Canyon Shapes による広域のうねりと Seaside Cliffs による非対称な断崖が加算合成され、砂漠特有の「侵食された台地と谷」を形成する
3. **崖周辺に集中する生態系** --- 樹木（オリーブ系）・岩・瓦礫はすべて崖オブジェクトの距離圏内に配置され、崖から離れた平坦部は完全な荒野となる

他バイオーム（メサ・サバンナ等）と比較して、テクスチャスタンプ数（2）、ディテールスタンプ数（1）、オブジェクトスタンプ数（2）のいずれも最少クラスであり、**要素の少なさ自体がデザイン**として機能している。

---

## 2. デザイン意図

### なぜこのパラメータ群が「砂漠」を成立させるのか

砂漠バイオームの本質は **「不在」のデザイン** にある。多くのバイオームが植生・テクスチャ・オブジェクトの多層的な重ね合わせで豊かさを表現するのに対し、砂漠はそれらを極限まで削ることで環境を語る。

**テクスチャの極端な単純化（2層のみ）：** SandFine がweight=1で全面を覆い、Mud がweight=0.099（約10%）でノイズ分布する。この「ほぼ砂一色」という構成は、現実の砂漠が鉱物組成の均一性から色相変化に乏しいという特性を反映している。泥パッチの存在はかつての水の痕跡を暗示し、完全に死んだ土地ではなく「干上がった」ニュアンスを加える。

**崖中心の空間構成：** 樹木・瓦礫はすべて崖オブジェクト（High Desert Cliffs）の子として配置され、距離制約で崖周辺26〜77mの帯に限定される。これは乾燥地帯において崖の影や裂け目に水分が集まり、その周辺だけに植生が成立するという生態学的リアリティに基づく。オアシスの概念を崖の距離圏として抽象化している。

**大きな標高差（約300m）：** SplineArea の Y 座標が +150 から -150 まで変動し、西が高台・東が低地という傾斜を形成する。砂漠が平坦であるという先入観を裏切り、峡谷と高原が共存する米国南西部的な景観を志向している。

## 3. HeightMap 構成

### ヒエラルキー構造

```
砂漠エリア [SplineContainer, SplineArea]
  +-- High Cliffs with Trees [FalloffOverride]
       +-- Clear Stamp [ClearStamp]
       +-- Canyon Shapes 27 (Height Stamp) [HeightStamp]
       +-- Seaside Cliffs 02 (Height Stamp) [HeightStamp]
```

### SplineArea（エリア境界）

| パラメータ | 値 | 意図 |
|---|---|---|
| closedMode | Area（閉領域） | 不定形の砂漠境界を閉じたスプラインで定義 |
| sdfRes | K512 | SDF 解像度512。バイオーム境界のフォールオフ精度 |
| maxSDF | 128 | SDF 最大距離。フォールオフの最大幅 |
| ノット数 | 34 | 複雑な多角形で自然な境界形状を実現 |

エリアの概算サイズは東西約750m x 南北約1200m。西側が高く（Y=+150付近）、東側が低い（Y=-150付近）大きな傾斜がある。

### FalloffOverride

| パラメータ | 値 | 意図 |
|---|---|---|
| フィルタタイプ | スプラインエリア（index=4） | 親スプラインの形状で全子スタンプを制限 |
| splineAreaFalloff | 0 | フォールオフ幅ゼロ |
| splineAreaFalloffBoost | 0 | ブーストなし |
| falloffRange | (1.0, 1.0) | エリア内=フル適用、エリア外=即座にゼロ |

falloffRange が (1.0, 1.0) であるため、スプライン境界での減衰は**極めて急峻**。砂漠と他バイオームの境界がハードカットされる設計であり、砂漠の「断絶的な環境変化」を表現している。

### ClearStamp

| パラメータ | 値 |
|---|---|
| clearTrees | true |
| clearDetails | true |
| clearObjects | true |

HeightStamp 適用前にエリア内の既存データを全消去し、砂漠固有のアセットだけで空間を再構築する。

### HeightStamp #1: Canyon Shapes 27

**役割:** 砂漠全体にうねるような微細な谷筋を形成する峡谷パターン。

| パラメータ | 値 | 設計意図 |
|---|---|---|
| テクスチャ | Canyon Shapes 27.png（Rowlan） | 峡谷パターンのハイトマップスタンプ |
| ブレンドモード | Add（加算） | 既存地形に高さを加算。SplineArea の傾斜に谷筋を重ねる |
| スケール XZ | 1180.11 m | エリア全体（750x1200m）をほぼカバーする巨大範囲 |
| スケール Y | 0.29 | 振幅を大幅に抑え、**なだらかな峡谷パターン**を生成 |
| Power | 1 | リニア（非線形変形なし） |
| Erosion | 0 | 侵食なし。テクスチャ原型をそのまま使用 |

**なぜこの設定か：** Y スケール 0.29 という極めて低い振幅は、砂漠の谷筋が風化と堆積で平坦化された「なだらかな渓谷」を表現する。急峻な V 字谷ではなく、砂が堆積した U 字型のワジ（涸れ川）的な形状になる。1180m 四方の巨大範囲は、谷筋パターンがエリア全体に均一に広がる必要があるため。

### HeightStamp #2: Seaside Cliffs 02

**役割:** 砂漠内の断崖・崖地形を形成する。片側が急峻、反対側がなだらかな非対称地形。

| パラメータ | 値 | 設計意図 |
|---|---|---|
| テクスチャ | Seaside Cliffs 02.tif（MicroVerse 同梱） | 海岸崖パターン。非対称な断面を持つ |
| ブレンドモード | Add（加算） | Canyon Shapes の結果にさらに崖を加算 |
| スケール XZ | 602.52 x 426.27 m（非正方形） | 崖を一方向に引き延ばした形状 |
| スケール Y | 0.53 | Canyon Shapes（0.29）の約1.8倍の起伏 |
| Y 軸回転 | 262.44度 | 崖の向きをエリアの傾斜に合わせて調整 |
| Power | 1 | リニア |
| Erosion | 0 | 侵食なし |

**なぜこの設定か：** 非正方形スケール（602 x 426m）は崖を方向性のある帯状の地形にし、砂漠の台地エッジやキャニオンウォールを表現する。Y スケール 0.53 は Canyon Shapes より高い起伏を与え、崖として視覚的に認識できる高さを確保する。262.44 度の回転は、SplineArea の西高・東低の傾斜方向と組み合わせて、崖面が特定方向を向くよう調整されている。

### 地形の合成順序

```
1. SplineArea の Y 座標変化（+150 ~ -150）→ 全体の大きな傾斜
2. Canyon Shapes 27（加算, Y=0.29, 1180m四方）→ なだらかな渓谷模様
3. Seaside Cliffs 02（加算, Y=0.53, 600x426m）→ 非対称な崖地形
```

3つのレイヤーが加算で重畳され、「傾斜した大地に谷筋が走り、その中に断崖がそびえる」という砂漠峡谷の地形が構築される。

## 4. テクスチャ構成

### 全体構造

砂漠エリアのテクスチャスタンプはわずか2つ。全バイオーム中で草原（1つ）に次いで少ない。この「情報量の少なさ」が砂漠の本質である。

### TextureStamp #1: SandFine（ベースレイヤー）

| パラメータ | 値 | 設計意図 |
|---|---|---|
| レイヤー | SandFine.terrainlayer | 細かい砂のテクスチャ。PureNature Oasis 同梱 |
| Weight | 1.0 | フルウェイト。エリア全面を砂で覆う |
| ノイズ | None | ノイズなし。均一な砂地を形成 |
| フォールオフ | グローバル, falloffRange (0.8, 1.0) | エリア全体に適用 |

**なぜノイズなしの weight=1 か：** 砂漠の地表は風による選別（ウィノウイング）で粒径が均一化されるため、マクロスケールでは均質な外観になる。ノイズを加えないことで「どこまでも同じ砂」という砂漠の広大さと単調さを表現する。weight=1 はこのレイヤーが他のすべてに優先する「デフォルトの大地」であることを意味する。

### TextureStamp #2: Mud（泥パッチ）

| パラメータ | 値 | 設計意図 |
|---|---|---|
| レイヤー | Mud.terrainlayer | 泥テクスチャ。PureNature Oasis 同梱 |
| Weight | 0.099 | 非常に低いウェイト。砂の上にうっすらと泥が混じる |
| ノイズタイプ | WormFBM | ワーム状FBM。有機的な流線形パターン |
| ノイズ周波数 | 2.96 | 低周波。大きなスケールの泥分布 |
| ノイズ振幅 | 7.92 | 高振幅。ノイズの存在感を強める |
| ノイズバランス | -0.38 | 負バランス。泥の出現をさらに抑制 |
| ノイズ空間 | スタンプ | スタンプローカル座標でノイズを評価 |
| フォールオフ | グローバル, falloffRange (0.8, 1.0) | エリア全体に適用 |

**なぜ WormFBM + 低ウェイトか：** weight=0.099（約10%）は泥テクスチャを「ほとんど見えないがゼロではない」レベルに抑える。WormFBM ノイズ（周波数2.96, 振幅7.92）は蛇行する流線パターンを生成し、これが「かつて水が流れた痕跡」として砂地に泥の筋を残す。負のバランス値（-0.38）はノイズの出力を負方向にオフセットし、泥の出現領域をさらに限定する。

MicroVerse の Top-4 Index/Weight 方式では、SandFine（weight=1.0）が圧倒的に支配し、Mud（weight=0.099 x ノイズ値）はノイズが正の値を取る狭い領域でのみ砂を上書きする。結果として、広大な砂地にまばらな泥跡が蛇行するという、水の乏しい砂漠の地表が再現される。

### テクスチャ分配の視覚的結果

```
SandFine 100% ──────────────────────────── ほぼ全域
    └── Mud ~10% (WormFBMパターン) ──── 蛇行する細い帯状に散在
```

2 レイヤーのみの極めてシンプルな構成だが、泥の WormFBM パターンが微妙なテクスチャ変化を加えることで、完全な単色にはならない。近づくと砂の中に乾いた泥跡が見えるという、控えめだが重要なディテールが実現されている。

## 5. 草花・ディテール構成

### 全体構造

砂漠エリアのディテールスタンプはわずか1つ。全バイオーム中で最少。

### DetailStamp #1: Malcomia_grass（乾燥草）

| パラメータ | 値 | 設計意図 |
|---|---|---|
| プロトタイプ | Malcomia_grass | 乾燥に強いマルコミア草。PureNature アセット |
| ノイズタイプ | Simple | 単純なグラディエントノイズ（Perlin 的） |
| ノイズ周波数 | 12.87 | 高周波。細かいスケールの密度パターン |
| ノイズ振幅 | 16.7 | 非常に高い振幅。密度の変動幅が極めて大きい |

**なぜ Simple ノイズ + 高振幅か：** Simple ノイズ（Perlin 系グラディエントノイズ）は FBM や WormFBM よりパターンが単純で、島状の密度分布を作る。周波数 12.87 は中〜高周波で、数十メートルスケールの草の群生パッチを生成する。振幅 16.7 は非常に大きく、ノイズの正領域で草が生え、負領域では完全に消失する。これにより「砂漠の中に点在する草のパッチ」という、現実の乾燥地帯で見られるスポット植生パターンが再現される。

**草種の選択意図：** Malcomia は地中海・中東原産の一年草で、乾燥環境に適応した低矮な草本。砂漠バイオームにオリーブ系樹木と組み合わせることで、地中海性乾燥気候の植生を一貫したテーマで表現している。

### 配置の空間パターン

ディテールスタンプは崖オブジェクトの子ではなく、独立した "Grass" グループに属している。ただし FalloffOverride の影響下にあるため、SplineArea 内に限定される。崖からの距離制約はなく、エリア全体にノイズパターンで分布するが、密度の高い領域はノイズの偏りに依存する。

## 6. 樹木構成

### 全体構造

砂漠エリアの TreeStamp は1つ。崖オブジェクトの子として配置され、崖周辺に限定される。

### TreeStamp #1: オリーブ系樹木

| パラメータ | 値 | 設計意図 |
|---|---|---|
| density | 7.72 | 高密度。崖周辺の限定領域に集中 |
| poisson | 1.647 | Poisson Disk 強度。均一すぎず自然な間隔 |
| seed | 16 | 乱数シード |
| occludeOthers / occludedByOthers | True / True | 双方向オクルージョン有効 |
| プロトタイプ数 | 8 | Olivetree1-5 + Olivebush1-3 |
| スケール height | (0.8, 1.2) | 20%のサイズバリエーション |
| スケール width | (0.8, 1.2) | 20%のサイズバリエーション |
| ノイズタイプ | None（ウェイトノイズ無効） | フィルタノイズのみ使用 |
| フィルタノイズ周波数 | 3.79 | 低〜中周波。大きなパッチ単位の密度変動 |
| フィルタノイズ振幅 | 20.73 | 高振幅。密度の変動幅が大きい |

**配置距離制約（ObjectStamp の子として）:**

| パラメータ | 値 | 意図 |
|---|---|---|
| minDistanceFromParent | 33.0m | 崖の直近を避ける（崖面には木が生えない） |
| maxDistanceFromParent | 63.4m | 崖から約60m以内。水分の到達範囲を模擬 |

**なぜ density=7.72 で距離制約付きか：** density 値自体は高い（7.72）が、崖から33〜63m の帯状領域にのみ配置されるため、実質的な樹木数はエリア全体に対して少ない。この設計は「乾燥地の崖の根元に水分が集まり、そこだけに樹木が群生する」というオアシス的な生態を表現している。density を高くして配置領域を狭めることで、局所的には密な樹林だがグローバルには疎、という砂漠のコントラストが生まれる。

**プロトタイプの構成：** Olivetree（5種）は高木、Olivebush（3種）は灌木。8種のバリエーションにより、同じオリーブ科でも高さ・形状の変化があり、単調なプランテーションではなく自然な群落に見える。スケール (0.8, 1.2) の控えめなランダム化は、乾燥ストレス下で成長が制限された樹木の均一性を反映する。

## 7. オブジェクト構成

### 全体構造

砂漠エリアのオブジェクトスタンプは2つ。崖岩（High Desert Cliffs）とその周囲の瓦礫（Dense Rubble）で構成される。崖が空間の骨格を形成し、瓦礫がその足元を飾る親子関係になっている。

### ObjectStamp #1: High Desert Cliffs（崖岩）

| パラメータ | 値 | 設計意図 |
|---|---|---|
| density | 0.48 | 低密度。巨大な崖は数を抑えて希少性を維持 |
| poisson | 2 | 強い Poisson Disk。崖同士の重なりを防ぐ |
| seed | 298 | 乱数シード |
| プロトタイプ数 | 3 | DesertHighCliff_0, 1, 2 |

**Randomization 設定:**

| パラメータ | 値 | 設計意図 |
|---|---|---|
| scaleX/Y | (1, 2) | 最大2倍のサイズバリエーション。巨大な崖壁を実現 |
| rotationY | (-180, 180) | 全方位ランダム回転。同じ崖に見えない |
| slopeAlignment | 0.04 | ほぼゼロ。斜面に追従せず**直立配置** |
| sink | (-10, -12) | 10〜12m 地中に埋没。崖の根元を地面に溶け込ませる |

**なぜ slopeAlignment=0.04 + sink=(-10,-12) か：** 崖岩は地形の法線に追従してはならない。slopeAlignment をほぼゼロにすることで、斜面上でも崖は垂直に立つ。しかし、単に垂直に置くと地面との接合部に隙間ができるため、sink で 10〜12m 深く沈めて根元を隠す。この「直立 + 深い沈み込み」の組み合わせは、「地面から突き出た岩盤」という砂漠の崖の見え方を再現する MicroVerse の重要なテクニックである。

**ノイズ設定:**

| パラメータ | 値 | 意図 |
|---|---|---|
| ノイズタイプ | WormFBM | 有機的な帯状パターンで崖の分布を制御 |
| 周波数 | 10 | 中周波。数十メートル単位の密度変動 |
| 振幅 | 0.21 | 低振幅。微妙な密度差。均一すぎない程度 |

### ObjectStamp #2: Dense Rubble（瓦礫）

| パラメータ | 値 | 設計意図 |
|---|---|---|
| density | 2.63 | 崖（0.48）の約5.5倍。崖周辺を瓦礫で埋める |
| poisson | 2 | 強い Poisson Disk。瓦礫同士の重なり防止 |
| seed | 225 | 乱数シード |
| プロトタイプ数 | 6 | RubbleDense_1/2/3 + RubbleSparse_1/2/3 |

**Randomization 設定:**

| パラメータ | 値 | 設計意図 |
|---|---|---|
| scaleX/Y | (0.8, 1.2) | 控えめなサイズ変動。崖より小さいスケール感を維持 |
| slopeAlignment | 1.0 | **斜面に完全追従**。地面に貼り付くように配置 |
| sink | (0, 0.2) | わずかな沈み込み。砂に半分埋もれた瓦礫を表現 |
| ノイズ | None | ノイズなし。配置は距離制約のみで決まる |

**配置距離制約（崖からの距離）:**

| パラメータ | 値 | 意図 |
|---|---|---|
| minDistanceFromParent | 26.4m | 崖の直下は避ける（崖メッシュと干渉するため） |
| maxDistanceFromParent | 76.6m | 崖から約77mの範囲に散布 |

**なぜ崖=直立 / 瓦礫=完全追従 か：** この対比が空間のリアリティを生む。崖は地殻変動で隆起した硬い岩盤であり重力に抗って直立する（slopeAlignment=0.04）。瓦礫はその崖から風化・崩落した破片であり、斜面に沿って転がり堆積する（slopeAlignment=1.0）。物理的に正しい挙動を slopeAlignment の値で表現している。

**Dense vs Sparse の使い分け：** RubbleDense（密な瓦礫）と RubbleSparse（疎な瓦礫）の6種がすべて weight=50 で等確率選択される。これは崖の崩落が局所的に密だったり疎だったりするランダム性を再現する。

### オブジェクト配置の空間関係

```
崖（density=0.48, 低密度で散在）
  |
  +-- 26.4m〜76.6m --> 瓦礫（density=2.63, 高密度で帯状に散布）
  +-- 33.0m〜63.4m --> 樹木（density=7.72, 最高密度で帯状に群生）
```

崖を起点とした同心円状の配置構造が見える。崖の直下 26m 以内は空白（メッシュ干渉回避）、26〜77m に瓦礫、33〜63m に樹木。瓦礫の範囲が樹木より広く、崖から最も遠い場所では瓦礫のみが点在する。

## 8. MapGenerator 再現パラメータ

### 高度マップ

| 要素 | MapGenerator での対応 | パラメータ目安 |
|---|---|---|
| SplineArea の傾斜（+150 ~ -150） | バイオーム配置時の基本高度勾配関数 | amplitude=300m, 低周波の線形〜緩やかな勾配 |
| Canyon Shapes 27（加算, Y=0.29, 1180m） | Ridged Noise（リッジノイズ）、谷の絶対値反転 | amplitude=0.03（正規化）, scale=1180, octaves=1 |
| Seaside Cliffs 02（加算, Y=0.53, 600x426m） | Directional Ridged FBM + Domain Warp | amplitude=0.05, scaleX=600, scaleZ=426, warp_angle=262度 |

**高度マップ生成のアプローチ:**

1. **ベース傾斜:** エリアの大きな標高差（約300m）は、バイオームマスク境界内で低周波の勾配関数を適用して再現する
2. **峡谷パターン:** Ridged Noise の絶対値反転で谷筋を生成。低振幅（amplitude ~0.03 相当）、大きなスケール（frequency: 1/1180 程度）で、砂に埋もれたなだらかなワジを表現
3. **崖地形:** 方向性のある Ridged FBM + ドメインワーピングで片側急峻・片側なだらかな非対称崖を再現。Y 軸の回転は warp 方向のオフセットで制御。振幅は峡谷の約1.8倍

### テクスチャ分配

| MicroVerse パラメータ | MapGenerator での対応 |
|---|---|
| SandFine (weight=1, ノイズなし) | デフォルトテクスチャインデックス。バイオーム内の全ピクセルにベースとして割当 |
| Mud (weight=0.099, WormFBM freq=2.96 amp=7.92 bal=-0.38) | WormFBM ノイズマスク付きサブテクスチャ。threshold = max(0, wormfbm(uv * 2.96) * 7.92 - 0.38) * 0.099 |

テクスチャ選択は極めてシンプルで、斜面フィルタや高度フィルタは使用されていない。純粋にノイズベースの泥分布のみ。

### ディテール（草花）配置

| MicroVerse パラメータ | MapGenerator での対応 |
|---|---|
| Malcomia_grass, Simple noise, freq=12.87, amp=16.7 | 乾燥草プロトタイプ。Perlin noise(uv * 12.87) * 16.7 で密度マスク生成。正領域のみ配置 |

密度マスク生成の擬似コード:
```
density_mask = saturate(perlin_noise(uv * 12.87) * 16.7)
// 振幅が大きいため、ノイズ値の正負で二値的に分かれる
// 正領域 → 密な草、負領域 → 草なし
```
