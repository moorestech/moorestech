# メサエリア 総合分析

## 1. バイオーム概要

メサ（Mesa）は、浸食によって形成された台地状の地形を特徴とする乾燥地帯バイオームである。平坦な頂上部、急峻な崖面、崩落した瓦礫の斜面という三層構造が視覚的なアイデンティティを形成する。MicroVerse 実装では、全7バイオーム中**最大のスタンプ数**（HeightStamp 1 + TextureStamp 4 + DetailStamp 20 + TreeStamp 5 + ObjectStamp 8 = 合計38スタンプ）を持ち、最も精密にレイヤリングされたバイオームである。

| 項目 | 値 |
|------|-----|
| ルートオブジェクト | `MicroVerse/メサエリア` |
| ワールド位置 | (1368.21, 61.64, 1345.02) |
| 所属テレイン | `Terrain_(1000.00, 0.00, 1000.00)` (size: 1000x600x1000, res: 1025) |
| 子ノード数 | 54（階層最大深度6） |
| 境界定義 | SplineContainer + SplineArea（31ノット、不整形多角形） |
| テクスチャセット | BK PureNature MesaDesert |

### ヒエラルキー構造

```
メサエリア [SplineContainer, SplineArea]
  └─ Big Mesa 02 [FalloffOverride]       ← 全スタンプの空間マスク
       ├─ Clear Stamp [ClearStamp]        ← 他バイオーム植生のリセット
       ├─ Big Mesa 01/                    ← 地形構成の中核グループ
       │    ├─ Cliffs 01/
       │    │    ├─ Stony Sand 02/        ← TextureStamp x4
       │    │    ├─ Thin Mesa with Strate/ ← ObjectStamp: Strate + Thin Mesa
       │    │    └─ Grass 04/
       │    ├─ Big Mesa/                  ← ObjectStamp: Big Mesa 配置
       │    └─ Thin Mesa with Strate/     ← ObjectStamp: Strate Mesa Sharp 01
       ├─ Mixed Cactus (with Grass)/      ← TreeStamp + DetailStamp
       ├─ Senita/                         ← TreeStamp（セニタサボテン）
       ├─ Wildflowers 01/                ← DetailStamp x4
       ├─ Wildflowers 04/                ← DetailStamp x4
       ├─ Opuntia/                        ← TreeStamp（ウチワサボテン）
       ├─ Boulders/                       ← ObjectStamp（岩塊）
       └─ Canyon Shapes 30 (Height Stamp) ← 唯一の HeightStamp
```

## 2. デザイン意図

メサバイオームのデザインは「**地質学的な時間の痕跡**」を表現することに集約される。以下の設計判断にそれが現れている。

### なぜスタンプ数が最多なのか

メサ地形は単一のノイズ関数では表現困難な、**不連続的な形状**（急峻な崖 + 平坦な台地）を特徴とする。MicroVerse はこの問題を「HeightStamp で緩やかな起伏を作り、ObjectStamp の3Dメッシュで急峻な崖面を補う」という二段構えで解決している。ObjectStamp が8つも必要なのは、メサの崖面・地層・瓦礫・岩塊という異なるスケールの地質要素をそれぞれ独立に制御するためである。

### レイヤリング戦略の背景

メサ地形を構成する要素は、以下の地質学的階層に対応している：

1. **地盤**（HeightStamp）: キャニオン形状の大地形。浸食で削られた峡谷と残存台地
2. **岩体**（ObjectStamp: Big Mesa, Thin Mesa, Strate）: 台地上に露出した岩層。堆積岩の水平層理を表現
3. **崩落堆積物**（ObjectStamp: Dense Rubble, Boulders）: 崖面から崩れ落ちた岩屑。重力の結果
4. **地表被覆**（TextureStamp）: 砂と礫のテクスチャ。風化と堆積の結果
5. **植生**（TreeStamp + DetailStamp）: 乾燥に適応したサボテン・枯草。過酷な環境の証

この階層は**下から上へ、大スケールから小スケールへ**と積み上げられ、地質学的に正しい順序で景観を構築する。

## 3. HeightMap 構成

### Canyon Shapes 30 -- 唯一の HeightStamp

メサエリアの地形高さを直接制御するスタンプは1つだけである。キャニオン（峡谷）形状のハイトマップテクスチャで台地状の起伏を生成する。

| プロパティ | 値 | 意図 |
|-----------|-----|------|
| テクスチャ | Canyon Shapes 30 (4096x4096, R16) | Rowlan/Gaia Stamps パッケージ。峡谷と台地の組み合わせ |
| mode | **Override（上書き）** | 既存地形を完全置換。メサの地形は他バイオームと独立 |
| blend | 1.0 | 完全適用 |
| power | 1.0 | リニア（非線形変形なし） |
| 実効ワールドスケール | 1388.70 x 36.00 x 1388.70 | テレイン1枚超をカバー、高さ36m |
| Falloff（自身） | ボックス、range (0.80, 1.00)、リニア | 外周20%でフェードアウト |
| Falloff（親: Big Mesa 02） | SplineArea、距離30m、ブースト10、イーズインアウト | スプライン境界でマスク |

### Transform の構造

HeightStamp は Big Mesa 02（FalloffOverride）の子であり、スケールが親子で乗算される：

| レベル | スケール | 役割 |
|--------|---------|------|
| Big Mesa 02（親） | (1011.19, 120.00, 1220.73) | エリア全体のバウンディングボックス |
| HeightStamp（子） | (1.37, 0.30, 1.14) | 親に対する相対比率 |
| **実効スケール** | **(1388.70, 36.00, 1388.70)** | 最終的なワールドサイズ |

### なぜ Override モードなのか

他バイオーム（草原・森林等）は Add や Max モードでノイズを既存地形に重ねることが多い。メサが Override を採用する理由は、**キャニオン形状は「削られた結果」であり、加算では表現できない**ためである。既存の緩やかな地形を一旦消して、峡谷パターンで置き換えるのが最も直接的な方法である。

### なぜ高さレンジが36mと控えめなのか

テレイン最大高さ600mに対して6%の起伏しかない。これは意図的な設計で、急峻な崖面は HeightStamp ではなく **ObjectStamp の3Dメッシュ**（Strate, Big Mesa, Thin Mesa）が担当する。HeightStamp は地形の大まかなうねり（台地の高低差）だけを生成し、視覚的なインパクトのある垂直の崖はオブジェクトで後付けする。

### SplineArea 設定

31個のノットで不整形な閉じた多角形を形成。西側〜南側に弧を描き、東側は開放されている。

| プロパティ | 値 |
|-----------|-----|
| closedMode | Area |
| sdfRes | K512 (512px) |
| maxSDF | 128.0 |
| 実効範囲 | エリア中心から約 +-630m |

### ClearStamp

FalloffOverride 直下で、エリア内の既存植生を全リセットする。

| プロパティ | 値 |
|-----------|-----|
| clearTrees / clearDetails / clearObjects | 全て True |
| filterType | Global（FalloffOverride 範囲内で一律適用） |
| falloffRange | (0.80, 1.00) |

## 4. テクスチャ構成

### 4層レイヤリングシステム

メサエリアのテクスチャは、地質学的に正しい順序で4層を重ねる。全て BK PureNature MesaDesert パッケージのアセットを使用。

| # | レイヤー | weight | フィルタ条件 | ノイズ | デザイン意図 |
|---|---------|--------|-------------|--------|-------------|
| 1 | **SandLarge** | 1.0 | なし | なし | ベースレイヤー。台地上の粗い砂地を全面に敷く |
| 2 | **SandFine** | 1.0 | slope: 2.3-19.5度, smooth: (6,6) | なし | 斜面専用。崖面の細かい砂が流れ落ちた表現 |
| 3 | **SandFine_gravel** | 0.548 | なし | WormFBM freq=14.29 amp=7.92 | 礫混じりの砂。高周波ノイズで細かい斑模様 |
| 4 | **SandFine_rubble** | 0.299 | なし | WormFBM freq=2.97 amp=15.51 | 岩屑混じりの砂。低周波・高振幅で大きなパッチ |

### なぜこの構成なのか

**第1層（SandLarge）**: weight=1.0、フィルタなしのベースレイヤー。メサ地帯全体を覆う粗い砂のテクスチャ。これが他の全レイヤーの「下地」となる。

**第2層（SandFine）**: slope フィルタ（2.3-19.5度）が鍵。水平面（0度）では適用されず、緩い〜中程度の斜面にのみ現れる。smoothness が (6,6) と広めに設定されており、斜面と平地の境界がなめらかにブレンドされる。崖面に堆積した細粒砂を表現するが、垂直に近い急崖（19.5度超）には適用しない -- そこは ObjectStamp の岩メッシュが覆うためである。

**第3層（SandFine_gravel）**: weight=0.548（約55%）で WormFBM ノイズ（freq=14.29）を使用。高周波数は**小さな礫が散在する細かいパッチ**を生み出す。WormFBM ノイズの蛇行パターンが、風で吹き寄せられた礫の自然な分布を模倣する。

**第4層（SandFine_rubble）**: weight=0.299（約30%）で WormFBM ノイズ（freq=2.97）を使用。第3層の約1/5の周波数で振幅は約2倍。これにより**大きな岩屑の堆積パッチ**が生まれる。崖の基部に堆積した崩落物のテクスチャとして機能する。

### 第3層と第4層のノイズ設計比較

| パラメータ | SandFine_gravel | SandFine_rubble | 比率 |
|-----------|----------------|-----------------|------|
| frequency | 14.29 | 2.97 | 4.8x |
| amplitude | 7.92 | 15.51 | 0.5x |
| weight | 0.548 | 0.299 | 1.8x |

周波数と振幅が逆相関している点に注目すべきである。高周波・低振幅（gravel）は細かく散在する礫、低周波・高振幅（rubble）は広く堆積する岩屑を表現する。これはフラクタルの自己相似性と同じ原理 -- **大きいものは少なく、小さいものは多い** -- に基づく自然な分布である。

## 5. 草花・ディテール構成

### 20スタンプの分析

メサエリアは20個の DetailStamp を持つ。これは全バイオーム中最多であり、乾燥地帯の疎らな植生を精密に制御するために多数のスタンプが必要になっている。

#### グループ別内訳

| グループ | スタンプ数 | プロトタイプ | ノイズ | デザイン意図 |
|---------|-----------|-------------|--------|-------------|
| Strate 下 | 4 | DryGrass2/3/4 | WormFBM freq=12.87 amp=16.7 | 地層岩の根元に生える枯草。WormFBM で蛇行する群落 |
| Strate Mesa Sharp 下 | 4 | DryGrass2/3/4 | なし, freq=12.87 amp=16.7 | メサ岩周辺の枯草。ノイズなしでより均一な分布 |
| Mixed Cactus 下 | 4 | DryGrass2/3/4 | WormFBM freq=12.87 amp=16.7 | サボテンの間に生える枯草 |
| Wildflowers 01 | 4 | WildflowersYellow1-4 | Simple freq=12.87 amp=16.7 | 黄色い野花。Simple ノイズでランダムな散在 |
| Wildflowers 04 | 4 | WildflowersYellow1-4 | WormFBM freq=25.84 amp=16.7 | 黄色い野花。高周波 WormFBM で密集パッチ |

#### DryGrass のパターン分析

12個の DryGrass スタンプのうち、8個は WormFBM ノイズ付き、4個はノイズなし。同一プロトタイプ（DryGrass2/3/4）に対して異なるノイズ設定の複数スタンプが存在する理由は、DetailStamp のマージ方式にある。

MicroVerse は同一プロトタイプの複数スタンプを **max（最大値）合成** する。WormFBM ノイズ付きスタンプは蛇行パターンで密度が 0 になる領域があるが、ノイズなしスタンプがベースラインとして下支えする。結果として、**蛇行パターンで密度が高い領域 + それ以外でも最低限の密度**という二段階の分布が実現される。

#### WildflowersYellow の二重配置

Wildflowers 01（Simple ノイズ）と Wildflowers 04（WormFBM freq=25.84）で同じ WildflowersYellow1-4 を配置している。Simple ノイズはランダムだが均一な散在を作り、WormFBM は蛇行する密集帯を作る。max 合成の結果、**蛇行する花の群落と、それを繋ぐ疎らな点在**が生まれ、現実の野花の分布パターンに近づく。

#### freq=12.87 / amp=16.7 の共通パラメータ

ほぼ全ての DetailStamp が freq=12.87, amp=16.7 という同じノイズパラメータを使用している。これはメサエリア全体で植生パターンのスケール感を統一するための設計判断である。周波数を統一することで、枯草と野花が同じ空間周期で変化し、バイオーム内の視覚的一貫性が保たれる。

## 6. 樹木構成

### 5スタンプの役割分担

メサエリアの「樹木」は全てサボテン類で、5つの TreeStamp で4種のサボテンを配置する。

| # | プロトタイプ | 数 | density | ノイズ | デザイン意図 |
|---|-------------|-----|---------|--------|-------------|
| 1 | Senita1-8 + Saguaro1-5 | 13 | 1.7 | Simple f=25.84 a=8 | 混合サボテン。低密度で大型種を疎らに配置 |
| 2 | Cacactus1-4 | 4 | **12.3** | なし | 小型サボテン。最高密度で地表を埋める |
| 3 | Senita1-8 | 8 | 6.8 | なし | セニタサボテン単独。中密度で均一配置 |
| 4 | Senita1-8 | 8 | 3.0 | Simple f=25.84 a=8 | セニタサボテン追加。ノイズで疎密を付与 |
| 5 | Opuntia1-4 | 4 | 8.0 | Simple f=25.84 a=8 | ウチワサボテン。高密度だがノイズで群落化 |

共通設定: poisson=1.647, height=(0.8,1.5), width=(0.8,1.5)

### なぜ Senita が2スタンプに分かれているのか

スタンプ#3（density=6.8, ノイズなし）はベースラインとして均一にセニタサボテンを配置する。スタンプ#4（density=3.0, Simple ノイズ）は追加の群落を重ねる。TreeStamp のマージもmax合成であるため、**均一なベース + ノイズによる密集帯**という DetailStamp と同じ二層パターンが適用されている。

### 密度の階層構造

| 種 | 合計密度 | 性質 |
|----|---------|------|
| Cacactus（小型） | 12.3 | 地被植物的。最も密に茂る |
| Opuntia（ウチワ） | 8.0 | 中型。パッチ状に群生 |
| Senita（柱状） | 6.8 + 3.0 = 9.8 | 中〜大型。均一+群落 |
| Saguaro（巨大） | 1.7（混合スタンプ内） | 最大型。最も疎ら |

この密度勾配は生態学的に正しい。小型の種ほど個体数が多く、大型の種ほど個体間距離が広い。サワロサボテン（Saguaro）が density=1.7 と極めて低いのは、実際のソノラ砂漠での分布密度を反映している。

### Simple ノイズ (f=25.84, a=8) の効果

3つのスタンプ（#1, #4, #5）が共通して Simple f=25.84, a=8 を使用する。高周波（25.84）は細かい空間変動を作り、振幅8は配置判定の weight に大きな揺らぎを与える。結果として、配置密度が局所的に大きく変動し、サボテンが「まばらにある場所」と「密集する場所」のコントラストが生まれる。

## 7. オブジェクト構成

### 8スタンプの階層的レイヤリング

メサエリアの ObjectStamp は8つで、全バイオーム中最多。地質学的な層序に対応した階層構造を持つ。

| # | オブジェクト | density | poisson | プロトタイプ数 | slopeAlign | ノイズ | Active |
|---|------------|---------|---------|--------------|------------|--------|--------|
| 1 | Strate（地層岩） | 0.80 | -- | 6 (Strate_0-5) | 0.605 | WormFBM | Yes |
| 2 | Dense Rubble（密瓦礫） | 5.79 | 0.21 | 3 (RubbleDense) | 1.0 | なし | Yes |
| 3 | Strate Mesa Sharp 01 | 3.41 | 0.17 | 5 | 0.605 | -- | **No** |
| 4 | Thin Mesa（薄メサ岩） | 0.80 | -- | 6 (ThinMesa_0-5) | 0.325 | WormFBM | Yes |
| 5 | **Big Mesa**（大メサ岩） | 4.41 | -- | 6 (BigMesa_0-5) | 0.325 | WormFBM a=3.34 | Yes |
| 6 | Strate Mesa Sharp 01 | 0.93 | -- | 5 | 0.605 | WormFBM | Yes |
| 7 | Dense Rubble（疎瓦礫） | 7.39 | 0.21 | 3 (RubbleSparse) | 1.0 | なし | Yes |
| 8 | Boulders（大岩） | 1.51 | -- | 5 (Boulders_1-5) | 1.0 | -- | Yes |

### 地質学的レイヤリングの意図

#### 骨格層: メサ岩体（#1, #4, #5, #6）

Strate（地層岩）、Thin Mesa（薄いメサ）、Big Mesa（大メサ）は台地の「骨格」を形成する。slopeAlign が 0.325〜0.605 と中間的な値であるのは、岩層が完全に水平（0.0）でも完全に地形追従（1.0）でもなく、**概ね水平だが微かに傾いた堆積岩層**を表現するためである。

- **Strate** (slopeAlign=0.605): 地層の露頭。地形にやや追従するが、水平層理を保つ
- **Thin Mesa** (slopeAlign=0.325): 薄い台地。よりフラットに近い配置
- **Big Mesa** (slopeAlign=0.325): 大きな台地岩塊。Thin Mesa と同じ追従度

#### 崩落層: 瓦礫（#2, #7）

Dense Rubble が2回登場するのは、**異なるプロトタイプ**を使い分けるためである。#2 は RubbleDense（密集した瓦礫塊）、#7 は RubbleSparse（散在する瓦礫）。いずれも slopeAlign=1.0 で完全に地形に追従する -- 崩落した瓦礫は重力で地面に張り付くからである。

poisson=0.21 という極めて小さな値に注目すべきである。Poisson Disk のジッター強度が低いということは、**グリッド状に近い均一な配置**になることを意味する。瓦礫は崖の基部に沿って帯状に堆積するため、ランダム感よりも均一な被覆が重要である。

#### 独立層: 大岩（#8）

Boulders は density=1.51 と低密度で、崩落した巨石を疎らに配置する。slopeAlign=1.0 で地面に密着し、scaleX=(0.5,0.8) で小さめにスケーリングされる。大きな岩が少数転がっている風景を作る。

### なぜ Strate Mesa Sharp 01 が2つあり、1つは無効なのか

#3（Active=No, density=3.41, poisson=0.17）と#6（Active=Yes, density=0.93）の2つが存在する。#3 は高密度だが無効化されている。これはイテレーションの痕跡 -- 開発中に密度を試行錯誤し、最終的に低密度の#6 のみを採用した結果と考えられる。#3 を残しているのは、将来の調整や比較のためだろう。

### density と poisson の設計パターン

| オブジェクト | density | poisson | 配置パターン |
|------------|---------|---------|------------|
| Strate / Thin Mesa / Big Mesa | 0.80〜4.41 | デフォルト | 適度にランダムな散在。岩体の自然な配置 |
| Dense Rubble (2種) | 5.79 / 7.39 | 0.21 | 高密度・低ジッター。均一に地面を覆う瓦礫 |
| Boulders | 1.51 | デフォルト | 低密度で疎らな巨石 |

density が高い順: RubbleSparse(7.39) > RubbleDense(5.79) > BigMesa(4.41) > StrateMesaSharp(3.41/0.93) > Boulders(1.51) > Strate/ThinMesa(0.80)

**大きなオブジェクトほど密度が低い**という原則が一貫している。瓦礫（小さい）は高密度で地面を覆い、メサ岩体（大きい）は低密度で点在する。

### WormFBM ノイズの役割

Strate, Thin Mesa, Big Mesa は WormFBM ノイズを使用して配置パターンに蛇行性を与えている。WormFBM は sin ベースのフローパターンを FBM で重ねたノイズで、**帯状・河川状のパターン**を生成する。これにより岩体が直線的ではなく、蛇行する尾根や崖に沿って配置される。Big Mesa の amplitude=3.34 は他より低く、大きな岩体の配置変動を抑えて安定感を出している。

## 8. MapGenerator 再現パラメータ

### HeightMap 再現方針

メサ地形のプロシージャル再現には、以下のアプローチが有効である。

#### キャニオン形状の生成

Canyon Shapes 30 テクスチャの特徴（深い溝と平坦な台地）を再現するには、**Ridged ノイズの反転 + 台地化関数**の組み合わせが最も近い。

```
mesa_height(x, z) = clamp(quantize(ridged_noise(x, z), steps=4), 0, 1) * amplitude
```

- **Ridged Noise**: `1 - abs(noise(x, z))` で鋭い尾根を生成
- **反転**: 尾根を谷に変換（キャニオンの溝になる）
- **Quantize（量子化）**: 高さ値を離散的な段階に丸めて台地のフラットな頂上を生成
- **amplitude**: 36m 程度（実効高さレンジに合わせる）

#### 高さパラメータ

| パラメータ | 推奨値 | 根拠 |
|-----------|--------|------|
| 高さレンジ | 36m | 実効 Y スケール |
| 台地化段数 | 3〜5段 | テクスチャ観察から推定 |
| ノイズ周波数 | 低〜中 | 大きな台地構造 |
| 浸食量 | 0（MicroVerse でも未使用） | HeightStamp に erosion=0 |

### テクスチャ再現方針

```
texture_weights(x, z):
  base     = SandLarge: 1.0 (全面)
  slope    = SandFine:  slope_filter(2.3-19.5度, smooth=6)
  gravel   = SandFine_gravel: worm_fbm(freq=14.29, amp=7.92) * 0.548
  rubble   = SandFine_rubble: worm_fbm(freq=2.97, amp=15.51) * 0.299
```

- 斜面フィルタの実装: `smoothstep(2.3-6, 2.3, slope) * (1 - smoothstep(19.5, 19.5+6, slope))`
- gravel/rubble は WormFBM ノイズの出力を weight として乗算

### 植生再現方針

| 種別 | 密度制御 | ノイズ |
|------|---------|--------|
| 小型サボテン (Cacactus) | density=12.3, ノイズなし | 均一配置 |
| ウチワサボテン (Opuntia) | density=8.0 | Simple f=25.84 a=8 |
| セニタサボテン (Senita) | density=6.8 (均一) + 3.0 (ノイズ) | Simple f=25.84 a=8 |
| サワロサボテン (Saguaro) | density=1.7 | Simple f=25.84 a=8 |
| 枯草 (DryGrass) | 12スタンプ合成 | WormFBM f=12.87 a=16.7 + ノイズなし |
| 黄花 (WildflowersYellow) | 8スタンプ合成 | Simple f=12.87 + WormFBM f=25.84 |

### オブジェクト配置再現方針

MapGenerator でメサの3Dオブジェクト配置を再現する場合の優先度：

1. **Big Mesa 岩体**: 最大の視覚的インパクト。低密度 (4.41) で WormFBM ベースの配置。slopeAlign=0.325
2. **Dense Rubble**: 高密度 (5.79/7.39) で均一配置。崖の基部の瓦礫層
3. **Strate / Thin Mesa**: 低密度 (0.80) の地層岩。WormFBM で蛇行配置
4. **Boulders**: 低密度 (1.51) の巨石。完全地形追従

### バイオーム境界ブレンド

| パラメータ | 値 | MapGenerator での対応 |
|-----------|-----|---------------------|
| SplineArea フォールオフ距離 | 30m | バイオーム境界のブレンド幅 |
| フォールオフブースト | 10.0 | 内側を強調する乗数 |
| イージング | EaseInOut | `f < 0.5 ? 2*f*f : 1 - pow(-2*f+2, 2)/2` |

境界から30m以内でブレンドが発生し、ブースト10によって境界近くまでメサの影響が強く残る。結果として、メサとの境界は比較的シャープな遷移になる。
