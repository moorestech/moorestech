# 岩石山エリア（Rocky Mountain）総合分析

## 1. バイオーム概要

岩石山エリアは MicroVerse シーン内で最も劇的な垂直スケールを持つバイオームである。火山山地ハイトマップによる急峻な岩山を中心に、巨大な砂漠断崖オブジェクト（スケール4〜6倍）が林立し、荒涼とした岩石砂漠の景観を構成する。植生は最小限に抑えられ、オリーブの木がまばらに点在する程度。全バイオーム中、地形の高度幅（278m）が最大であり、遠景からの視認性が高いランドマークとして機能する。

**基本情報:**

| 項目 | 値 |
|---|---|
| ルート Position | (904.00, 75.90, 743.00) |
| SplineArea ノット数 | 36（不規則多角形） |
| ワールド範囲（概算） | X: 418〜1487, Z: 0〜1314 |
| HeightStamp 数 | 3 |
| TextureStamp 数 | 2 |
| DetailStamp 数 | 1 |
| TreeStamp 数 | 2 |
| ObjectStamp 数 | 3 |

**ヒエラルキー構造:**

```
岩石山エリア [SplineContainer, SplineArea]
  └─ Big Desert Cliffs [FalloffOverride: SplineArea参照]
       ├─ Clear Stamp [ClearStamp]
       ├─ Sand Covered Mud
       │    ├─ Ground [TextureStamp] — SandFine
       │    └─ Ground [TextureStamp] — Mud
       ├─ Desert Cliffs
       │    └─ Desert Cliffs [ObjectStamp] — 巨大断崖

       │              │    └─ Grass [FalloffOverride] ※非アクティブ
       │              └─ Trees [TreeStamp] — Olivetree
       │                   ├─ Plants [DetailStamp] — Malcomia_grass
       │                   └─ Trees [TreeStamp] — Olivebush

```

## 2. デザイン意図

岩石山エリアのデザインは「荒涼とした巨大岩石砂漠」を目指している。以下の3つの柱でこの印象を実現する。

### ドラマチックな垂直スケール

メインの HeightStamp（T_HeightMap4k）は Y スケール 2.32 で、高度幅 278m を生成する。これは全バイオーム中最大であり、遠景から見たときにスカイラインを支配するランドマークとなる。火山山地のハイトマップにより、鋭い尾根と深い谷が複雑に交差する山容が得られる。

### 巨大オブジェクトによる断崖の演出

最大の特徴は ObjectStamp の第1スタンプ（Desert Cliffs）である。scaleX/scaleY が (4, 6) に設定されており、通常の4〜6倍のスケールで砂漠断崖メッシュを配置する。これにより、ハイトマップだけでは表現しきれない垂直に切り立った崖面や岩のオーバーハングが生成される。この巨大スケール設定は全バイオーム中で唯一であり、岩石山エリア固有のビジュアルアイデンティティを形成している。

### 植生の極端な抑制

テクスチャは SandFine（砂）をベースに Mud が僅か（weight 0.099）だけ混ざる2層構成。草花は Malcomia_grass の1種のみ。樹木も Olivetree と Olivebush の2スタンプだけで、密度は中程度。この極端な植生抑制が、地形とオブジェクトの存在感を際立たせている。

## 3. HeightMap 構成

岩石山エリアは3つの HeightStamp で構成される。メインの岩山地形が1つ、接続エリアの遷移地形が2つという3層構造。

### 3.1 T_HeightMap4k — メインの岩山地形

エリア全体を覆う基礎地形。火山山地のハイトマップで急峻な岩肌を生成する。

| プロパティ | 値 |
|---|---|
| テクスチャ | `T_HeightMap4k`（VolcanoMountains） |
| 解像度 / フォーマット | 2048x2048 / R16 |
| Position | (943.58, 37.26, 724.61) |
| LocalScale | (1.50, **2.32**, 1.22) |
| LossyScale | (1606.57, 277.95, 1606.57) |
| CombineMode | Override |
| 高度範囲 | 37.26m〜315.21m（**278m幅**） |
| Falloff | Box, Range (0.8, 1.0), ノイズなし |
| Power / Erosion / Twist | 1 / 0 / 0 |

**Y スケール 2.32 の意味:** LossyScale.y が 277.95m となり、テクスチャの 0〜1 の高さ値が 278m の高度差に展開される。他バイオームの HeightStamp と比較してこの高度幅は突出して大きい。Power=1, Erosion=0 であるため、ハイトマップテクスチャの形状がそのまま反映される。

### 3.2 Terraced Cliffs 05 — 接続エリアの段丘崖

接続エリア内に配置された段丘状の崖。隣接バイオームとの高度遷移を緩和する。

| プロパティ | 値 |
|---|---|
| テクスチャ | `Terraced Cliffs 05` |
| 解像度 / フォーマット | 4096x4096 / R16 |
| Position | (615.33, 86.72, 430.67) |
| LossyScale | (466.82, 28.73, 466.82) |
| CombineMode | Override |
| 高度範囲 | 86.72m〜115.45m（28.7m幅） |
| Falloff | Range, Range (0.8, 1.0) |

接続エリアの FalloffOverride（Range, 0.56〜1.0）内で動作する。467m四方・高度幅29mと小規模で、段丘状の棚と壁が交互に現れる地形パターンを生成する。

### 3.3 Broken Lands 01 — 接続エリアの荒野丘陵

もう1つの接続スタンプ。崩壊した丘陵地形で不規則な起伏を生成する。

| プロパティ | 値 |
|---|---|
| テクスチャ | `Broken Lands 01`（Gaia Stamps） |
| 解像度 / フォーマット | 4096x4096 / R16 |
| Position | (587.94, 62.83, 656.55) |
| LossyScale | (445.83, 36.96, 445.83) |
| CombineMode | Override |
| 高度範囲 | 62.83m〜99.79m（37.0m幅） |
| Falloff | Box, Range (0.8, 1.0) |

Position が北寄り（Z=656）に配置されており、Terraced Cliffs 05（Z=430）とペアでメインスタンプとの間の高度遷移を分担する。

### 地形構成のまとめ

```
高度 (m)
315 ┬─────── メインスタンプ上限
    │  ████  T_HeightMap4k（278m幅）
    │  ████  火山山地の鋭い尾根・深い谷
 75 ┼─────── メインスタンプ下限付近
    │
115 │  ▓▓    Terraced Cliffs 05（29m幅）段丘状
 87 │
100 │  ░░    Broken Lands 01（37m幅）崩壊丘陵

    └─────── 基底
```

全スタンプが Override モードのため、ヒエラルキーの順序（兄弟順）で後のスタンプが前のスタンプを上書きする。接続エリアのスタンプはメインスタンプより後に処理され、境界遷移を上書きで制御する。

## 4. テクスチャ構成

テクスチャ構成は全バイオーム中で最もシンプル。砂地のベースに泥がごく僅かに混ざる2層のみ。

### 4.1 SandFine — ベースレイヤー

| プロパティ | 値 |
|---|---|
| TerrainLayer | SandFine |
| Weight | 1.0 |
| Noise | None |

エリア全体を覆うベースの砂地テクスチャ。ノイズなしの weight=1 で均一に適用される。

### 4.2 Mud — サブレイヤー

| プロパティ | 値 |
|---|---|
| TerrainLayer | Mud |
| Weight | **0.099** |
| Noise Type | WormFBM |
| Noise Frequency | 2.96 |
| Noise Amplitude | 7.92 |

Weight が 0.099 と極めて低いのが特徴。WormFBM ノイズで変調されるが、低い weight のため泥テクスチャはほとんど目立たない。WormFBM（ワーム型 fBm）は蛇行した有機的なパターンを生成し、砂地の中に僅かな泥の筋が現れる効果を狙っている。

**テクスチャ適用の計算:** MicroVerse の FilterSet では `最終ウェイト = DoFilters() * _Weight * Falloff` として計算される。Mud の場合、WormFBM ノイズが DoFilters() の weightNoise として作用し、0〜1 の変動にさらに weight=0.099 が乗算されるため、最終的な適用強度は最大でも約 0.1 程度に留まる。

**砂漠エリアとの比較:** 砂漠エリアも同一の SandFine + Mud（weight=0.099, WormFBM f=2.96 a=7.92）構成を持つ。両エリアは地面テクスチャを共有しており、砂漠〜岩石山の連続性を視覚的に示唆している。

## 5. 草花・ディテール構成

ディテール配置は全バイオーム中で最小限。1種の草スタンプのみ。

### 5.1 Malcomia_grass

| プロパティ | 値 |
|---|---|
| Prototype | Malcomia_grass |
| Noise Type | None |
| Noise Frequency | 12.87 |
| Noise Amplitude | 16.7 |

Malcomia_grass は乾燥環境向けの低い草。ノイズが None に設定されているため、FilterSet の weightNoise による密度変調はなく、Falloff とフィルタ条件（高度・斜面等）のみで配置が制御される。freq=12.87, amp=16.7 はパラメータとして存在するが、ノイズタイプが None のため無視される。

**岩石山エリアの植生戦略:** ClearStamp がエリア内の既存植生（Tree, Detail）を事前にクリアし、その上に最小限の Malcomia_grass だけを配置する。これにより、隣接する植生豊富なバイオーム（森林・林など）の草が岩山内部に侵入することを防ぎつつ、完全に無植生にはしない自然な外観を維持している。

## 6. 樹木構成

樹木は Olivetree（オリーブの木）と Olivebush（オリーブの低木）の2スタンプ。密度は中程度で、岩山の合間にまばらに配置される。

### 6.1 Olivetree（高木）

| プロパティ | 値 |
|---|---|
| Prototypes | Olivetree1〜5（5種） |
| Density | 3.74 |
| Noise Type | FBM |
| Noise Frequency | 3.79 |
| Noise Amplitude | 20.73 |

FBM ノイズ（3オクターブ fBm）で配置密度を変調する。frequency=3.79 はやや低周波で、大きな塊で木が群生する領域と疎な領域が交互に現れるパターンを生成する。amplitude=20.73 は高く、ノイズの影響が強いことを示す（weightNoise は `1 + noise` として計算されるため、高い振幅は密度の空間変動を大きくする）。

### 6.2 Olivebush（低木）

| プロパティ | 値 |
|---|---|
| Prototypes | Olivebush1〜3（3種） |
| Density | 3.19 |
| Height Range | (1, 1)（固定サイズ） |
| Noise Type | None |

ノイズなしの均一な密度で配置される。高さ範囲が (1, 1) で固定されており、サイズのばらつきがない。Olivetree よりやや低い密度だが、ノイズがないためより均一に分布する。

**配置ヒエラルキーの意味:** TreeStamp は ObjectStamp（Desert Rocks）の子として配置されている。MicroVerse のヒエラルキーでは親の Falloff が子に継承されるため、岩石が配置された領域と同じ範囲内に樹木が配置されることになる。SDF オクルージョンが有効であれば、岩の近くへの配置回避も可能。

## 7. オブジェクト構成

オブジェクト構成は岩石山エリアの最大の特徴であり、3つの ObjectStamp が階層的に配置されている。特に第1スタンプの巨大スケール設定は全バイオーム中で唯一の手法。

### 7.1 Desert Cliffs（巨大断崖）

| プロパティ | 値 |
|---|---|
| Prototypes | DesertCliff_0〜4（5種） |
| Density | 0.25 |
| Scale X/Y | **(4, 6)** |
| Slope Alignment | 0.056 |
| Sink | (3, 6) |
| Noise | なし |

**スケール (4, 6) の異常性:** 通常のオブジェクトスタンプでは scaleRange は (0.8, 1.2) や (1, 2) 程度。この (4, 6) は桁違いに大きく、断崖メッシュが通常の4〜6倍のサイズで配置される。これにより、プレイヤーが見上げるような巨大な岩壁が生成される。

**低密度 + 巨大スケール:** density=0.25 は非常に低い。ObjectStamp の instanceCount は `Round(512 * density^2 * terrainScale^2)` で計算されるため、0.25 の密度では配置数はごく少数になる。少数の巨大断崖が点在するという構成。

**Sink (3, 6):** 3〜6m 地形に沈み込ませることで、断崖の根元が地面に自然に接続する。巨大スケールに合わせて沈み込み量も大きい。

**Slope Alignment 0.056:** ほぼ地形法線に追従しない（垂直に配置）。断崖は垂直に立つべきなので、地形の傾斜にほとんど影響されない設定。

### 7.2 Desert Cliffs（小型断崖）

| プロパティ | 値 |
|---|---|
| Prototypes | DesertCliff_0〜4（5種） |
| Density | 1.76 |
| Scale X/Y | (1, 2) |
| Slope Alignment | 0 |
| Noise Type | WormFBM |
| Noise Frequency | 5.32 |
| Noise Amplitude | **-5.54** |

**負の振幅 (a=-5.54) の意味:** MicroVerse のノイズは `(NoiseFunc(uv * freq + offset) - balance) * amplitude` として計算される。amplitude が負の場合、ノイズ出力が反転する。WormFBM が通常 0〜1 付近の正の値を返すとすると、負の振幅により weightNoise の結果は `1 + (正の値 * -5.54)` = 負の値になりやすい。DoFilters() の最終結果は `saturate()` で 0〜1 にクランプされるため、多くの領域で密度が 0 に落ちる。

つまり、負の振幅ノイズは **排除ゾーンの生成** に使われている。WormFBM の蛇行パターンに沿って断崖が消え、残った隙間にだけ小型断崖が配置される。この手法により、巨大断崖（第1スタンプ）と小型断崖（第2スタンプ）の配置が重複せず、自然な疎密パターンが得られる。

### 7.3 Desert Rocks（岩石）

| プロパティ | 値 |
|---|---|
| Prototypes | DesertRock_1〜9（9種） |
| Density | 6.68 |
| Scale X/Y | (0.8, 1.2) |
| Slope Alignment | 1.0 |
| Noise | なし |

9種のバリエーションを持つ岩石を高密度で配置する。density=6.68 は高く、instanceCount の計算上かなりの数の岩が散布される。slopeAlignment=1.0 で地形法線に完全追従し、斜面に沿って自然に転がっているような配置になる。スケールは標準的な (0.8, 1.2) で、大小のばらつきは小さい。

### オブジェクト階層構造の設計意図

```
Desert Cliffs (巨大, density=0.25, scale=4-6)     ← 少数の巨大ランドマーク
  └─ Desert Cliffs (小型, density=1.76, scale=1-2)  ← 負ノイズで間引かれた中型断崖
       └─ Desert Rocks (標準, density=6.68)           ← 高密度の地面の岩
```

この3層階層は大→中→小のスケール階層を形成し、フラクタル的な岩石景観を生み出す。ヒエラルキーの親子関係により、親の Falloff が子に継承され、大きな断崖が存在する領域内にのみ小さな岩が配置される。

## 8. MapGenerator 再現パラメータ

岩石山エリアを MapGenerator で再現する際の設計指針と推奨パラメータ。

### 8.1 地形生成

| 要素 | 推奨手法 | パラメータ指針 |
|---|---|---|
| メイン地形 | 高オクターブ fBm | 高度幅 250〜300m 相当の振幅。6〜8 オクターブで鋭い尾根 |
| 段丘遷移 | ステップ関数（floor による量子化） | 高度幅 30m 程度。4〜6段の棚状構造 |
| 荒野遷移 | 低振幅の fBm | 高度幅 30〜40m。メインノイズとは異なるシードで不規則感を追加 |
| 境界フォールオフ | 距離ベースの smoothstep | メインエリアは急峻な切り替え (1,1)、接続エリアは漸進的減衰 (0.56,1) |

**地形が全バイオーム中最も急峻である点が最重要。** fBm の lacunarity を高く（2.5〜3.0）、gain を大きめ（0.55〜0.6）に設定し、高周波成分を多く残すことで鋭い起伏を実現する。

### 8.2 テクスチャ割り当て

| レイヤー | 条件 | 備考 |
|---|---|---|
| SandFine | デフォルト（全面） | weight=1.0 |
| Mud | WormFBM ノイズ、weight=0.1 以下 | freq=3, amp=8 程度の蛇行パターン |

砂漠エリアと共通のテクスチャ設定を使えば、バイオーム間の視覚的連続性が保たれる。

### 8.3 植生配置

| 種別 | 配置条件 | 密度 |
|---|---|---|
| Malcomia_grass | エリアマスク内、フィルタ通過時のみ | 低い。ノイズ変調なし |
| Olivetree | FBM ノイズ (f=3.8, a=20) による群生パターン | density=3.7 |
| Olivebush | 均一配置 | density=3.2 |

### 8.4 オブジェクト配置

| 種別 | 密度 | スケール | 特記事項 |
|---|---|---|---|
| 巨大断崖 | 0.25（極低） | 4〜6 倍 | sink=3〜6m。slopeAlign ほぼ 0（垂直配置） |
| 小型断崖 | 1.76 | 1〜2 倍 | WormFBM 負振幅 (a=-5.5) で排除ゾーン生成 |
| 岩石 | 6.68（高） | 0.8〜1.2 倍 | slopeAlign=1.0（完全地形追従）。9種バリエーション |

### 8.5 重要な再現ポイント

1. **巨大スケールオブジェクトが景観の核心。** ハイトマップだけでは岩石山の迫力は出ない。4〜6倍スケールの断崖オブジェクトが視覚的インパクトの大部分を担う
2. **負の振幅ノイズによる排除ゾーン。** 第2断崖スタンプの WormFBM a=-5.54 は、蛇行パターンで配置を間引く高度なテクニック。MapGenerator では「ノイズ値が閾値以下なら配置しない」というフィルタで同等の効果を得られる
3. **3層のオブジェクト階層。** 大→中→小のフラクタル的スケール構成が自然さの鍵。各層を独立のパスで生成し、親の Falloff を子に継承させる
4. **ClearStamp による植生排除。** 岩山エリア内に他バイオームの植生が残らないよう、生成前にクリアする仕組みが必要
5. **接続エリアの段丘 + 荒野丘陵。** メインの岩山（278m）から周辺地形（30〜40m）への急激な落差を、2つの接続スタンプで緩和する。FalloffOverride (0.56, 1.0) の漸進的減衰がブレンドの鍵

### 8.6 砂漠エリアとの差分

砂漠エリアと岩石山エリアはテクスチャ構成が同一だが、以下で明確に差別化されている:

| 項目 | 砂漠エリア | 岩石山エリア |
|---|---|---|
| 地形高度幅 | 中程度 | **278m（最大）** |
| 断崖スケール | 1〜2 倍 | **4〜6 倍** |
| 断崖の種類 | DesertHighCliff（高い砂漠崖） | DesertCliff（砂漠断崖） |
| 地面オブジェクト | RubbleDense/Sparse（瓦礫） | DesertRock（岩石）9種 |
| 樹木密度 | density=7.72（1スタンプ8種） | density=3.74+3.19（2スタンプ） |
| 負振幅ノイズ | なし | あり（a=-5.54） |
