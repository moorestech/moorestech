## 概要

`difit` は git diff を GitHub 風 UI でブラウザ表示する常駐 Web サーバー。`npx difit` で起動する。このスキルは差分範囲の指定方法と、常駐サーバーをセッションをブロックせずに起動する手順をまとめる。

## 前提条件

- Node.js / `npx` が利用可能（difit 自体のインストールは不要、`npx --yes difit` で都度取得される）。

## 差分範囲の指定

`difit [commit-ish] [compare-with] [options]`

| やりたいこと | コマンド |
|---|---|
| HEAD（最新コミット）の差分 | `difit` |
| 特定コミット単体の差分 | `difit <commit>` |
| 2点間の差分（A と B の間） | `difit <A> <B>` |
| 未コミットのワーキングツリー変更 | `difit .` |
| ステージ済み変更 | `difit staged` |
| ある基点以降の「コミット済み＋未コミット」を全部 | `difit . <base>` |
| GitHub の PR をレビュー | `difit --pr <PR-URL>` |

**「今回のセッションの変更を全部見たい」場合は `difit . <base>` を使う。** `<base>` はセッション開始時点のコミット（＝最初に作ったコミットの親 `git rev-parse <first-commit>~1`）。`.` がワーキングツリーを指すので、その後に積んだコミットも未コミット変更もまとめて1画面に出る。コミット範囲だけを指す `difit <HEAD> <base>` だと未コミット分が漏れる。

主なオプション:

- `--include-untracked` — 未追跡（新規）ファイルも差分に含める。新規ファイルを作った直後は付ける。
- `--no-open` — ブラウザを自動で開かない。
- `--port <n>` — 希望ポート（既定 4966、使用中なら自動繰り上げ）。

## 起動手順

1. 差分範囲を決める（上表）。新規ファイルがあれば `--include-untracked` を付ける。
2. 既に difit が動いていれば先に止める（下記 Gotchas 参照）。
3. **必ずバックグラウンドで起動する**（`run_in_background: true`）。difit は終了しない常駐サーバーなので、フォアグラウンドで叩くとセッションが固まる。

   ```
   npx --yes difit . <base> --include-untracked
   ```

4. 数秒待ってから出力ファイルを読み、`difit server started on http://localhost:<port>` の行で実際の URL とポートを確認してユーザーに伝える。ポートは使用中だと自動で繰り上がるため、出力を読むまで確定しない。
5. difit はブラウザを自動で開く（`--no-open` 指定時を除く）。

## Gotchas

- **difit は一度起動すると止まらない常駐サーバー。** フォアグラウンドで実行するとセッションがブロックされる。必ず `run_in_background: true` で起動する。
- **difit 自身の `--background` / `--keep-alive` フラグは `npx` プロセスを返さない。** これらは difit サーバーの挙動オプションであって、起動元の `npx` ラッパーは子プロセスを掴んだまま戻らない。フラグの有無に関わらず**必ず** harness の `run_in_background: true` で起動すること。フォアグラウンドで叩くと（`--background` 付きでも）difit サーバー自体は立つのにコマンドが戻らずセッションが固まる（実例：約20分スタックし手動解除が必要になった）。起動後は `sleep` で待たず、出力ファイルまたは `curl -s -o /dev/null -w '%{http_code}' http://localhost:<port>` で応答を確認する。
- **再起動・差分範囲の変更時は先に旧サーバーを止める。** `pkill -f difit` を実行してから新しい範囲で起動し直す。止めずに再起動すると別ポートに増殖し、古い差分のタブが残って紛らわしい。
- **ポートは固定ではない。** 既定 4966 が使用中なら 4967, 4968... と自動で繰り上がる。起動コマンドの出力（バックグラウンドの出力ファイル）を読んで実ポートを確認してから URL を案内する。
- **未コミット変更を見せたいなら commit-ish に `.` を使う。** コミット同士の範囲指定では未コミット分が表示されない。「手元の差分も含めて」と言われたら `.` を使う。
- **新規ファイルは `--include-untracked` を付けないと出ない。** git 未追跡ファイルは既定で差分に含まれない。
