# create-infographic-light — ペラ1 HTML 図解（コメント機能付き）

Markdown 等の技術文書を、単一の `index.html`（インライン CSS + vanilla JS、ビルド・CDN・外部依存なし、file:// で開いて動く）に図解する。create-infographic の 5 フェーズ（原稿レビュー・Mermaid・React/Vite・品質ゲート）を省いた軽量版で、**レビュー用コメント機能だけは本家同等を必ず組み込む**。

## 本家との使い分け

| 条件 | 使うスキル |
|---|---|
| 早く読みやすくしたい・レビューでコメントを集めたい | このスキル |
| 原稿の文章品質ゲート・Mermaid 図・TOC・タップ拡大が必要な公開級の図解 | create-infographic |

## ワークフロー

### 1. 入力の確認

- **本文ソース**: 図解対象の文書ファイルパス。これが唯一の正で、**事実の追加・改変・創作は禁止**（構成の並べ替え・表/カード/コールアウトへの起こしは自由）。
- 対象読者やページ構成の指定があれば控える。なければ「文書をレビューする人向け」をデフォルトとする。

### 2. 配置先を決める

`/tmp/infographic-{topic}/index.html`。**作業中の git リポジトリの中に作らない**（`git status` を汚すため。本家と同じ規約）。

### 3. sonnet サブエージェントへ委譲する

実装はメインエージェントが書かず、Agent ツール（model: sonnet）に委譲する。プロンプトに以下を全て含める:

1. **本文ソースの絶対パス** と「事実の追加・改変・創作は禁止」の明記
2. **テンプレートの絶対パス**: このスキルの `assets/template.html` を Read させ、**CSS とコメント機能 JS は verbatim 維持（リファクタ・省略・セレクタ変更禁止）、コンテンツ部分だけ差し替え**と指示する。コメント機能を自作・再移植させない — テンプレートは動作確認済みの完成品で、README の必須要件（selectionchange デバウンス・hover:none フォールバック・safe-area・16px 入力欄）を全て織り込み済み
3. **ページ要件**: 縦スクロール単一ページ / 白背景ライトミニマル / 絵文字不使用 / 日本語 / `overflow-wrap: anywhere` / 390px で body 幅 = viewport 幅
4. **構成の型**（文書に合わせて調整可）: リード 3 行 → Before/After 等の対比 → 変更点・本論 → トレードオフのコールアウト → 一覧テーブル → References（`<code>` でパス列挙）
5. **コメント可能ブロックの指定**: 対比・図解・テーブル・重要コールアウトを `.figure[data-label]` で包む。**`data-label` は図の中身を自然言語で書く**（「〜の図」「〜の比較」だけの外枠ラベルは禁止。コピーされた Markdown だけ読む第三者に中身が伝わる文にする）
6. **CONFIG の固有化**: `STORAGE_KEY` と `COPY_TITLE` をページ固有値に変えさせる（プレースホルダのまま・他ページと同値だと localStorage でコメントが混ざる）
7. **自己チェック**: タグバランス確認 + `<script>` 本文を抽出して `new Function()` で SyntaxError が無いことを node で確認させる

### 4. 検証する

- ブラウザで実挙動確認（claude-in-chrome が使えるなら: 表示・コメント追加・すべてコピー）。**使えない環境（拡張未接続等）なら、grep で主要要素（title / data-label 数 / STORAGE_KEY 固有化）を確認した上で「自動検証はできていない」と正直に報告する**。動くはず、とは言わない。
- テスト用に付けたコメントは localStorage から消してから完了する。

### 5. 完了報告

`open {出力パス}` でユーザーのブラウザに開き、出力パス・コメント機能の使い方（選択→追加 / 図の右上ボタン / 左下パネル / すべてコピー）・検証でやれたこと/やれなかったことを報告する。レビュー用途なら「コメントを付けて『すべてコピー』の Markdown を貼り返してもらえれば反映する」まで案内する。

## Gotchas

- **コメント機能をゼロから書かせない**。React 版（create-infographic の comment-feature.tsx）からの再移植も不要。`assets/template.html` に vanilla JS 移植済みの完成品が入っている。毎回の再実装は劣化とモバイル要件の取りこぼしを生む。
- **テンプレートの HTML コメント内に `-->` を含む説明文を書かない**。外側のコメントが早期終了して以降の本文が壊れる（テンプレート抽出時に実際に踏んだバグ）。
- **Before/After 対比で Before を創作しない**。ソース文書にある例から機械的に導出できる差分（例: 行末アンカーの有無）だけを付与する。
- **`data-label` の外枠ラベル禁止**は本家と同じ理由 — ラベルは図コメントの引用としてコピー出力に載るため、それだけ読んで中身が分かる必要がある。
- 検証不能を握りつぶさない。ブラウザ確認ができなかったらその旨を完了報告に含める（「HTML 構文チェックのみ」等）。

## Bundled assets

- `assets/template.html` — ページ骨格 + コメント機能（CSS/vanilla JS、動作確認済み）+ 各コンポーネント（対比・コードブロック・コールアウト・テーブル・References）の最小サンプル。冒頭の使い方コメントと `<script>` 内 CONFIG 注記（STORAGE_KEY / COPY_TITLE）を参照。サブエージェントには必ずこれを Read させて使わせる。
